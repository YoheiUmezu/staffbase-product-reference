# ニュースの追加と管理のための役割

**URL**: https://support.staffbase.com/hc/ja/articles/115003790291
**取得日**: 2026-05-02 18:48:03 JST

## 内容

役割の違いを知っておくことは重要です。ニュースの追加と管理には、コンテンツ特有の役割が通常は十分です。しかし、いくつかのユースケースでは、追加のチャンネルを設定したり、Spacesを使用した分散環境で作業するために、より強力な役割が必要になることがあります。

役割の詳細
ユーザー
グループ管理とコンテンツの可視性設定を使用することで、特定のユーザーのニーズに非常に特化したコンテンツをターゲットにすることが可能です。たとえば、特定の場所や興味グループに対してです。ターゲットにされていないユーザーはニュースを表示しません。
ターゲットユーザーは、自分または所属するグループのために公開されたコンテンツを見る権利があります。コンテンツを見ることに加えて、ユーザーはコメント、いいね、共有を通じてそれに関与することができます。さらに、インタラクティブなニュースチャンネルでは、ユーザーはそのチャンネルの他のメンバーに記事、更新、写真を投稿することができます。
チャンネル著者 = 貢献者
チャンネル著者は、単一のチャンネルに貢献するために明示的に設計された役割です。この役割は、多くのユーザーが参加し、相互作用することを望むインタラクティブなチャンネルに最適ですが、チャンネルの設定にアクセスしたり変更したりする能力はありません。
チャンネルエディター
チャンネルエディターは、ユーザーが1つのチャンネルを促進することを可能にする役割です。これは、他のユーザーのニュース投稿を編集、公開、非公開にすることや、自分のニュース投稿を貢献することを意味します。また、チャンネルエディターはチャンネルの設定にアクセスし、チャンネル設定に特定の変更を加えることができます。
グループ管理者
グループ管理者は、グループメンバーを追加および削除し、グループ設定を調整する権利を持っています。グループ管理者は、コンテンツではなくユーザーに焦点を当てています。
スペース管理者
スペース管理者は、コンテンツ管理のためのスペース全体の管理権限を持っています。
システム全体のエディター
システム全体のエディターは、通常のエディターよりもはるかに多くの権限を持ち、プラットフォーム内のシステム全体のニュースチャンネルやニュース投稿にアクセスし、促進することができます。システム全体のエディターは、新しいチャンネルを追加することはできません—それは管理エディターができることです。
管理エディター
管理エディターは、プラットフォーム内のシステム全体のニュースチャンネルやニュース投稿を管理する完全な権利を持っています。管理エディターは新しいチャンネルを追加することができ、これがこの役割をシステム全体のエディターと区別する権利です。
管理者
管理者は、コンテンツ管理を含むシステム全体の管理権限を持っています。
Staffbaseは、管理者がユーザー管理やプラットフォームの設定にアクセスできるため、コンテンツ管理にこの役割を使用することを推奨していません。
