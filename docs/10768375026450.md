# API対応プロファイルを従業員のメール連絡先として同期する

**URL**: https://support.staffbase.com/hc/ja/articles/10768375026450
**取得日**: 2026-05-02 18:43:44 JST

## 内容

親管理者
リリースされた連絡先APIは、従業員メールのユーザー情報を、任意のREST API対応ディレクトリからメール連絡先に同期し、カスタム配信リストを作成することを可能にします。
組織が内部連絡先情報を保存するためにMicrosoft Entra IDやWorkday以外のサードパーティツールを使用している場合、従業員メールの連絡先APIを活用して新しい統合を構築できます。APIの使用経験があることが推奨されます。
同期された連絡先情報は、定期的に自動的に最新の状態に保たれます。
インポートした連絡先を使用して、ウェブアプリでカスタム配信リストを作成します。
オプションで、サードパーティディレクトリから組織の既存の配信リストを直接同期するためのカスタムフィールドを作成できます。
連絡先APIを使用して、異なるユーザーデータのコレクションを従業員メールに同期したり、組織内の異なるインスタンスから同期したりするために、複数の統合を設定できます。
注意: この機能は、従業員メールウェブアプリから直接メールを送信する場合にのみ利用可能です。アドインや拡張機能をインストールし、メールクライアントから追跡されたメールを送信する場合、この情報は関連しません。
組織がStaffbaseのイントラネットおよび/またはアプリソリューションを従業員メールと統合しており、連絡先がすでにアプリ/イントラネットユーザーとして保存されている場合、Staffbaseのアプリ/イントラネットからユーザーを同期することで時間を節約できます。
前提条件
APIの使用経験があること。
Staffbaseは、統合の構築と維持を支援するために開発者やIT専門家のチームを持つことを推奨します。
従業員メールの親管理者アカウントを持っていること。
組織がメールを送信するためにメールウェブアプリを使用していること。
組織がStaffbaseのイントラネットおよび/またはアプリソリューションを従業員メールと統合しており、連絡先がすでにアプリ/イントラネットユーザーとして保存されている場合、Staffbaseのアプリ/イントラネットからユーザーを同期することで時間を節約できます。
従業員メール連絡先APIの使い方
メールウェブアプリにサインインします。
連絡先 > インポートに移動します。
連絡先APIタイルをクリックします。
紹介ページが開きます。
これは、組織に既存の連絡先API構成がない場合にのみ表示されます。
クリックして続行します。
連絡先フィールドタブが開きます。
七つのデフォルトフィールドの名前と、連絡先のデータをマッピングするために使用される可能性のある既存のカスタムフィールドを確認します。
オプションで、既存のカスタムフィールドを編集するか、カスタムフィールドを追加をクリックして、連絡先に関する追加データをマッピングします。
ヒント: 既存のサードパーティ配信リストに連絡先が属しているかどうかのデータを同期するには、同期したい各リストを表すカスタムフィールドを作成します。
統合が設定されると、これらのフィールドを使用して連絡先をフィルタリングし、カスタム動的配信リストを作成して、既存の同期リストに属する人だけを含めることができます。
カスタムフィールドに変更を加えた後、保存して更新をクリックします。
オプションで、APIを使用したときのフィールド名がどのように見えるかを確認するには、サンプルJSONをダウンロードをクリックできます。
APIキータブをクリックして、APIキーを生成します。
統合が実行されると、このキーがサードパーティアプリケーションを従業員メールと認証するために使用されます。
警告: APIキーは機密情報として扱い、他の人と共有しないでください。このキーは、親管理者と同じレベルのアクセス権を持って連絡先を表示および変更するために使用できます。
連絡先APIの開発者ドキュメントを使用して、従業員メールをサードパーティディレクトリに接続します。
APIの使用経験があることが推奨されます。
追加の役立つ情報
カンマ区切り値 (CSV) ファイルを介してEmail 連絡先データをアップロードする
Microsoft Entra ID (以前の Azure AD) ユーザーをEmail 連絡先として同期する
Workday ユーザーをEmail 連絡先として同期する
Staffbase アプリ/イントラネット ユーザーをEmail 連絡先として同期する
