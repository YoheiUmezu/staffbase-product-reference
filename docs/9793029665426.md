# プッシュ通知のための本番SSL証明書の準備

**URL**: https://support.staffbase.com/hc/ja/articles/9793029665426
**取得日**: 2026-05-02 18:49:30 JST

## 内容

この証明書は、Staffbaseサーバーからアプリに送信されるプッシュ通知のための安全な接続を保証します。
Apple Developerアカウントで、証明書、ID、およびプロファイル > 識別子 > アプリIDに移動します。
作成したアプリIDをクリックします。
このアプリIDに関連する設定が表示されます。
編集をクリックします。
プッシュ通知の設定に移動します。
本番SSL証明書のセクションで、証明書を作成をクリックします。
次のステップに進むには、証明書署名要求（CSR）が必要です。Apple Developerインターフェースを離れ、CSRを作成した後に戻ってきます。
本番SSL証明書の署名をリクエストする
次のステップにはMacが必要です。他のオペレーティングシステムで作業している場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーに連絡して、さらなるサポートを受けてください。
Macで、キーチェーンアクセスプログラムを開きます。
メニューバーで、キーチェーンアクセス > 証明書アシスタント > 証明書機関から証明書をリクエストに移動します。
証明書アシスタントが開きます。
ダイアログに詳細を入力します。
続行をクリックします。
リクエストファイルを保存するフォルダーを選択します。
完了をクリックします。
本番SSL証明書の作成
本番SSL証明書の作成を開始したApple Developerアカウントに戻ります。
続行をクリックします。
先ほど生成したCSRファイルを選択します。
続行をクリックします。
新しく作成された本番SSL証明書をダウンロードします。
完了をクリックします。
新しい証明書のある証明書の概要が表示されます。
Staffbaseに送信するための証明書ファイルの作成
次のステップにはMacが必要です。他のオペレーティングシステムで作業している場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーに連絡して、さらなるサポートを受けてください。
Staffbaseは、アプリに証明書ファイルを統合します。キーチェーンアクセスを使用すると、必要なファイルを簡単にエクスポートできます。
Macでキーチェーンアクセスプログラムを開きます。
メニューバーで、キーチェーンアクセス > アイテムのインポートに移動します。
先ほど作成した証明書を選択します。
キーチェーンアクセスウィンドウで、証明書の概要に移動します。
証明書の前にある矢印をクリックします。
プライベートキーを含む2行目が表示されます。
メニューバーで、ファイル > アイテムのエクスポートに移動します。
ファイル形式として「.p12」を選択します。
名前を入力します。名前はpush_で始まる必要があります。
このファイルと次のステップで作成される他のファイルを収集するフォルダーを選択します。
保存をクリックします。
後でStaffbaseに送信するためにファイルを保持します。
