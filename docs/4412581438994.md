カスタムドメインを有効にする
従業員メールは、リンクのクリック、ファイル、調査イベントを追跡するためにリダイレクトURLを使用し、メールのオープントラッキングコンポーネントにも同様のシステムを使用します。
これは、組織が使用しているホスティングに応じて、以下の一般的なドメインのいずれかで行われます:
米国ホスティング - https://s.bl-1.com または https://s2.bl-1.com
EUホスティング - https://es.de.bl-1.com
このドメインをカスタマイズして、組織のブランディングにより適したものにすることができます。これにより、内部フィッシングトレーニングプログラムに関連する潜在的な問題を軽減するのにも役立ちます。
これはどのように見えるか？
トラッキングピクセルコンポーネント（.gifファイル）もこのカスタムドメインの下にホストされます。
設定手順
ITチームは、以下の簡単な手順でカスタムドメインを設定できます:
CNAMEレコードを作成し、Bananatag（現在の従業員メール）を指すようにします。
サブドメインを選択して作成したら、ホスティングの場所に応じて以下のいずれかを指すようにします:
米国ホスティング: custom-domain-2.bananatag.com
EUホスティング: custom-domain.de.bananatag.com
完了したら、CNAMEの詳細を持ってStaffbaseに連絡し、正しく設定されていることを確認します。
Amazon Web Services（ACM経由）から公開証明書を発行できるようにするための追加のCNAMEレコードを提供します。
この証明書によりHTTPSが可能になります。
証明書はDNSレコードを削除することでいつでも取り消すことができます。
公開証明書に関する詳細はこちらをクリックしてください。
2つ目のレコードが追加されると、カスタムドメインが利用可能になります。
ヒント
custom-domain-2.bananatag.comやcustom-domain.de.bananatag.comを自分のリンクに置き換えないでください。むしろ、custom-domain-2やcustom-domain.deが作成するCNAMEのとして設定されていることを確認してください。
内部DNSレコードが維持されている場合、組織は追跡リンクを使用する際に「ページを表示できません」というエラーを回避するために、これらのレコードを更新する必要があります。
特殊文字を含まない限り、好きなドメイン名を選択できます。特殊文字はURLで機能します。
サブドメイン: サブドメインの使用を強く推奨します。トラフィックを当社のトラッキングサービスにルーティングするためには、指定されたドメインがDNSレコード（CNAME）で構成される必要があります。トップレベルドメイン（TLD）を選択する場合は、メール追跡目的のために明示的に意図されている必要があります。
例: トラッキングのために、GIF画像へのリクエストを使用します。example.comのようなTLDを使用する場合、example.comは当社のサービスにリダイレクトされる必要があります。そうしないと、example.com/tracking.gifやその他のパスのバリエーションが正しい場所に到達せず、機能しません。そのため、残りのドメインセクションに影響を与えずに、当社のサービスに簡単に制御しリダイレクトできるサブドメインを使用することを強く推奨します。
証明書: すべてのトラッキングリクエスト（GIF画像）は、標準HTTPSを使用して実行されます。安全な通信を確保するために、サブドメインのためにAWSから公開SSL証明書を登録します。さらに、公開証明書を検証するためのCNAMEレコードを顧客に提供します。
