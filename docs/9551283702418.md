# ニュースの投稿統計におけるトラフィックソース

**URL**: https://support.staffbase.com/hc/ja/articles/9551283702418
**取得日**: 2026-05-02 18:46:56 JST

## 内容

ニュースの投稿統計とリーチ分析は、従業員が投稿にどのように関与しているかについての洞察を提供します。これらの数値は視覚的に魅力的で、消費しやすい形で提示され、データに基づいたより良い意思決定を行うのに役立ちます。
トラフィックソースは、ユーザーが特定のニュース投稿やコンテンツにどのようにナビゲートしたかを説明します。すべてのトラフィックソースには、ユーザーがニュース投稿をクリック、開く、読み取る方法があります。たとえば、ユーザーがモバイルアプリで投稿に関するプッシュ通知を受け取り、それを開くと、プッシュ通知はトラフィックソースと見なされます。特定のトラフィックソースからの各クリックは、この指標にカウントされます。これらは、従業員がニュース投稿に至るまでの具体的な詳細と分析を提供する貴重な指標です。
トラフィックソース (A-Z)
ブックマーク
ユーザーは、ブックマーク機能を使用してニュース投稿をブックマークし、後で読むために保存したり、参照したりできます。ユーザーがブックマークを開くと、投稿に直接移動します。この場合、ブックマークはトラフィックソースとしてカウントされます。
コメント
ユーザーは、他のニュース投稿のコメント内に別のニュース投稿へのリンクを残すことができます。ユーザーがコメント内で共有されたリンクをクリックすると、ニュース投稿が開きます。この場合、コメントはトラフィックソースとしてカウントされます。
コメント通知
ユーザーは設定で、自分のニュース投稿に残されたコメントや他のニュース投稿に残したコメントの通知を受け取るオプトインができます。ユーザーがコメント通知をクリックすると、ニュース投稿が特定のコメントに直接開きます。このアクションによりニュース投稿が開かれるため、コメント通知はトラフィックソースとしてカウントされます。
デジタルサイネージ
Staffbaseプラットフォームでは、ニュースを大画面に表示し、スキャン可能なQRコードを含めることができます。ユーザーがQRコードをスキャンすると、モバイルアプリの新しい投稿に直接移動します。この場合、デジタルサイネージはトラフィックソースとしてカウントされます。
メール通知
メール通知は、Staffbaseユーザー管理と同期されたメールを持つユーザーに利用可能です。ニュース投稿を最初に公開する際やその後にメール通知を送信することができます。ユーザーは同じニュース投稿に対して複数のメール通知を受け取ることができます。ユーザーがメールに含まれるニュース投稿を開くと、メール通知はトラフィックソースとしてカウントされます。
リンク
寄稿者や編集者は、コンテンツエディタのリンクオプションを使用して、Staffbaseプラットフォーム全体の他の記事やコンテンツにニュース投稿を含めることができます。ユーザーが他のコンテンツ内のリンクをクリックすると、リンクはトラフィックソースとしてカウントされます。
リンクプラグイン
リンクプラグインは、ニュース投稿への直接リンクを含めることができます。ユーザーがリンクをクリックして開くと、リンクプラグインはトラフィックソースとしてカウントされます。複数のプラグインが同じ投稿へのリンクを持つことができ、各プラグインのクリックはすべてカウントされます。
リンクウィジェット
リンクウィジェットには、ニュース投稿へのリンクを含めることができます。ユーザーがリンクをクリックして開くと、リンクウィジェットはトラフィックソースとしてカウントされます。複数のウィジェットが同じ投稿へのリンクを持つことができ、各ウィジェットのクリックはすべてカウントされます。たとえば、Q1製品戦略に関する投稿が、部門ページの4つの異なるリンクウィジェットで使用されている場合、すべてのウィジェットからのクリックがカウントされます。
ニュースフィード
ニュースフィードは、ユーザーがStaffbaseアプリまたはイントラネットのニュースページでニュース投稿を表示する方法です。外部製品から取り込まれ、表示されるコンテンツは別途カウントされます。ニュースフィードは、ユーザーが元のフィードから投稿をクリックして開くと、トラフィックソースとしてカウントされます。
ニュースレター (従業員メール)
ニュース投稿は、従業員メールのニュースレター内のコンテンツブロックとして使用できます。読者がメールニュースレター内のリンクをクリックすると、投稿が開きます。この場合、ニュースレター (従業員メール)は、ユーザーが投稿リンクを開くとトラフィックソースとしてカウントされます。
投稿が複数のメールニュースレターに含まれる可能性があり、この場合、すべてのニュースレターが指標としてカウントされます。たとえば、HR WeeklyとCEO Monthlyの両方のニュースレターに、営業部長からの最近のニュース投稿へのリンクが含まれています。両方のニュースレターからのクリックがカウントされます。
ニュースフィードウィジェット
ニュースフィードウィジェットは、アプリまたはイントラネットのページに配置してニュースフィードを表示できます。ニュースフィードウィジェットは、ユーザーがウィジェット内の投稿プレビューをクリックしてニュース投稿を開くとトラフィックソースとしてカウントされます。投稿が複数のニュースフィードウィジェットに表示される可能性があり、この場合、すべてのニュースフィードウィジェットが指標としてカウントされます。たとえば、2つの異なるオフィスの場所ページに、ニュースフィードウィジェットを介してグローバルCEOニュースチャンネルが含まれています。両方のウィジェットからのクリックがカウントされます。
ニュースステージウィジェット
ニュースステージウィジェットは、アプリまたはイントラネットのページに配置して、ニュースページまたはチャンネルから選択されたコンテンツを表示できます。ニュースステージウィジェットは、ユーザーがウィジェット内の投稿プレビューをクリックしてニュース投稿を開くとトラフィックソースとしてカウントされます。投稿が複数のウィジェットに表示される可能性があり、この場合、すべてのニュースステージウィジェットが指標としてカウントされます。たとえば、3つの異なる部門ページに、グローバルHR更新ニュースチャンネルがニュースステージウィジェットを介して含まれています。すべてのウィジェットからのクリックがカウントされます。
通知センター
新しいコンテンツは、アプリまたはイントラネットの通知センターに表示されます。この場合、通知センターはトラフィックソースとしてカウントされます。ユーザーは通知センターに行き、投稿を複数回開くことができます。
その他
その他は、ソースが正確に追跡できない場合にトラフィックソースとしてリストされます。たとえば、まだ追跡できないサードパーティプラットフォームを介して送信された場合です。
プッシュ通知
プッシュ通知はモバイルアプリで利用可能で、ユーザーのモバイルに新しい投稿通知を直接表示します。投稿を最初に公開する際やその後にプッシュ通知を送信できます。ユーザーは同じニュース投稿に対して複数のプッシュ通知を受け取ることができます。ユーザーが通知から投稿を開くと、プッシュ通知はトラフィックソースとしてカウントされます。
QR
アプリやイントラネットとは別に、ニュース投稿へのリンクを生成するためのQRコードを作成できます。ユーザーがQRコードをスキャンすると、モバイルアプリ内の新しい投稿に直接移動します。この場合、QRはトラフィックソースとしてカウントされます。
RSSフィード
RSSフィードは、外部ウェブサイトからのニュースコンテンツをプラットフォーム内に直接表示します。ユーザーが投稿をクリックして開くと、RSSフィードはトラフィックソースとしてカウントされます。
検索
ニュース投稿は、コンテンツタブの下にある検索結果に表示されます。ユーザーが検索結果内の投稿をクリックすると、そのニュース投稿が開きます。この場合、検索はトラフィックソースとしてカウントされます。
SharePoint
Staffbaseプラットフォームからのニュース投稿は、SharePointとのNews Central統合を通じてリンクできます。ユーザーが統合内のリンクを開くと、SharePointはトラフィックソースとしてカウントされます。
共有
共有機能を使用すると、ユーザーは事前に設定された共有リンクを介して投稿を送信できます。リンクが共有されると、ユーザーはそれをクリックしてニュース投稿を直接開くことができます。この場合、共有はトラフィックソースとしてカウントされます。
スタジオ
ニュース投稿はスタジオからも開くことができます。ユーザーがスタジオからリンクをクリックしてニュース投稿を直接開くと、スタジオはトラフィックソースとしてカウントされます。
Teams
アプリとイントラネットからのニュース投稿は、Microsoft Teams内のSharePoint統合を通じてNews Centralにリンクできます。ユーザーが統合を介してリンクを開くと、Teamsはトラフィックソースとしてカウントされます。
ユーザープロファイル
ユーザーが貢献したりコメントしたりしたすべてのニュース投稿は、ユーザープロファイルページにリストされます。ユーザーがリストされたニュース投稿の1つをクリックすると、ユーザープロファイルはトラフィックソースとしてカウントされます。
URL
ニュース投稿は、ユーザーが直接URLリンクを受け取ることで開くことができます。ユーザーが直接リンクからニュース投稿を開くと、URLはトラフィックソースとしてカウントされます。たとえば、同僚がチャット機能を通じて別のユーザーに興味深いニュース投稿のURLを送信します。
