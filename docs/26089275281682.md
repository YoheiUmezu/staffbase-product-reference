# 機能展開戦略の概要

**URL**: https://support.staffbase.com/hc/ja/articles/26089275281682
**取得日**: 2026-05-02 18:45:54 JST

## 内容

Staffbaseは、メジャーリリースおよびマイナーリリースにおいて、機能の展開方法として3つの異なるタイプを提供しています。各機能に適した展開（ロールアウト）戦略は、プロダクトチームと関係するステークホルダーによって決定されます。新しい機能や更新、その展開戦略については、常にStaffbaseリリースプロセスを通じて通知されます。
3つの展開戦略は次のとおりです：
段階的ロールアウト
有効化
自動ロールアウト
段階的ロールアウト
段階的ロールアウトでは、Staffbaseが機能を段階的に顧客ベースへリリースします。この方法により、Staffbaseプロダクトチームはリスクを管理し、少人数のユーザーグループでパフォーマンスを監視できます。また、更新が管理された環境でどのように動作するかを確認できます。重大な問題が検出されなければ、ロールアウトは段階的に拡大されます。この段階的なアプローチにより、すべてのユーザーに更新が届いた際にスムーズな体験が保証され、必要に応じて迅速な調整が可能です。段階的ロールアウトの詳細（タイミングや範囲など）は、機能やその他の要因によって異なります。
有効化
有効化ベースの展開では、機能はプラットフォームに配信されますが、Staffbaseによってお客様の組織向けに有効にされるまで非アクティブのままです。つまり、機能は存在しますが、有効にされるまでプラットフォームで利用できません。これにより、どの機能をプラットフォームで有効にするかを管理できます。有効化を希望する場合は、カスタマーサクセスマネージャーまたはStaffbaseサポートにご連絡ください。すべてのStaffbaseベータ機能は、有効化によってのみ利用可能です。
自動ロールアウト
自動ロールアウトでは、リリース時に機能が自動的にプラットフォームで有効化されます。これは、すでにプラットフォームで有効化されている機能や製品にのみ適用されます。まだ有効化されていない新しい機能や製品は、リリース時に自動的にロールアウトされることはありません。自動ロールアウトにより、プラットフォームを最新の状態に保ち、セキュリティや安定性の向上、新機能へのアクセスが可能になります。
