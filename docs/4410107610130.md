# Microsoft 365 アプリケーションの権限の確認

**URL**: https://support.staffbase.com/hc/ja/articles/4410107610130
**取得日**: 2026-05-02 18:55:59 JST

## 内容

Microsoft 365 アプリケーションの権限は、組織を代表して付与する必要があります。その後、組織の Microsoft Entra ID (旧 Azure Active Directory) にアプリの参照が作成されます。また、アプリの権限を確認し、組織を代表して管理者の同意を付与することも可能です。
前提条件
スタジオで、次のいずれかを構成しています:
Microsoft Connection
Microsoft Plugin
次のいずれかの役割を持つ Azure アカウントがあります:
グローバル管理者
特権ロール管理者
クラウドアプリケーション管理者
アプリケーション管理者
ユーザーは、アプリケーションに権限を付与する権利を含むカスタムディレクトリロールが割り当てられている場合、テナント全体の同意を付与する権限を持つこともできます。
Microsoft Entra ID で、エンタープライズアプリケーションに移動します。
権限を管理したい Microsoft コンテンツのタイプを選択します:
Microsoft Search and Widgets
Staffbase プラットフォームとの Microsoft 接続を確立するには、使用したアプリに基づいて次のいずれかを選択します:
デフォルトの Azure アプリ: Staffbase Integrations を選択します
カスタム Azure アプリ: 作成したカスタムアプリを選択します
Microsoft 365 Widgets
Microsoft 365 Widgets の権限を構成するには、Staffbase MS365 Widgets を選択します。
Microsoft 365 Calendar Plugin
Microsoft Calendar Plugin の権限を構成するには、Staffbase MS365 Calendar Plugin を選択します
Microsoft 365 Files Plugin
Microsoft Calendar Plugin の権限を構成するには、Staffbase MS365 Files Plugin を選択します。
権限の下で、Microsoft アプリケーションの権限を確認できます。
内部コミュニケーションプラットフォームのためにアプリケーションを有効にする必要がある場合は、組織を代表して権限を付与するをクリックします。
