# iOS配布証明書の作成

**URL**: https://support.staffbase.com/hc/ja/articles/115003458931
**取得日**: 2026-05-02 18:48:06 JST

## 内容

iOS配布証明書は、アプリのコードがアプリに署名した組織から発信され、変更されていないことを保証します。
iOS配布証明書の準備
Apple Developerアカウントにサインインし、証明書、ID、およびプロファイル > 証明書 > プロダクションに移動します。
新しい証明書を追加します。
プロダクションタイプの証明書を設定し、App StoreおよびAd Hocを有効にします。
続行をクリックします。
次のステップに進むには、証明書署名要求（CSR）が必要です。Apple Developerインターフェースを離れ、CSRを作成した後に戻ってきます。
証明書の署名の準備
次のステップにはMacが必要です。他のオペレーティングシステムで作業している場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーに連絡して、さらなるサポートを受けてください。
MacでKeychain Accessプログラムを開きます。
メニューバーで、Keychain Access > 証明書アシスタント > 証明書機関から証明書を要求に移動します。
証明書アシスタントが開き、要求の情報を検証する証明書機関に提出される証明書署名要求（CSR）を作成します。
ダイアログにApple IDとメールアドレスを入力します。
続行をクリックします。
CSRファイルを保存するフォルダーを選択します。
完了をクリックします。
iOS配布証明書の最終化
証明書の作成を開始したApple Developerアカウントに戻ります。
続行をクリックします。
先ほど生成したCSRファイルを選択します。
続行をクリックします。
証明書が作成され、署名されます。
新しく作成された証明書をダウンロードします。
完了をクリックします。
新しい証明書を含む証明書の概要が表示されます。
Staffbaseに送信する証明書ファイルの作成
次のステップにはMacが必要です。他のオペレーティングシステムで作業している場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーに連絡して、さらなるサポートを受けてください。
StaffbaseはiOS配布証明書ファイルを使用してアプリに署名します。Keychain Accessを使用すると、必要なファイルを簡単にエクスポートできます。
MacでKeychain Accessプログラムを開きます。
メニューバーで、Keychain Access > アイテムのインポートに移動します。
先ほど作成した証明書を選択します。
Keychain Accessウィンドウで、証明書の概要に移動します。
証明書の前にある矢印をクリックします。
プライベートキーを含む2行目が表示されます。
メニューバーで、ファイル >アイテムのエクスポートに移動します。
ファイル形式として「.p12」を選択します。
このファイルと次のステップで作成される他のファイルを収集するフォルダーを選択します。
名前を入力します。名前はdistribution_で始まる必要があります。
保存をクリックします。
後でStaffbaseに送信するためにファイルを保持します。
