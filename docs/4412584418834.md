# 従業員メールWebアプリはなぜ連絡先のメールをマッピングする必要があるのか？

**URL**: https://support.staffbase.com/hc/ja/articles/4412584418834
**取得日**: 2026-05-02 18:58:59 JST

## 内容

製品名に関する注意:
2022年以降、BananatagはStaffbaseの従業員メール製品です。
Staffbaseの従業員メールのいくつかの重要な側面は、一時的にBananatagの名前を保持しています。たとえば、app.bananatag.comなどです。
実質的に、BananatagとStaffbaseの従業員メールは同義です。
従業員メール Webアプリを使用し、CSVファイルをインポートしてディレクトリに連絡先を追加する際、以下のメッセージが表示されることがあります:
メール フィールドはまだマッピングする必要があります。Bananatagは、連絡先を一意のメールアドレスで識別するため、インポートしたフィールドの1つは、続行する前にメール フィールドにマッピングされている必要があります。
このメッセージは、CSVをインポートする際にフィールドをマッピングしているときにのみ表示され、Bananatagフィールド名のメールがスキップされた場合に表示されます。
メールフィールドは、従業員メールに連絡先をインポートするために必要な最小限の情報です。システムは各連絡先のメールアドレスを一意の識別子として使用し、重複を解決します。
インポートしたい各連絡先にはメールアドレスを含める必要があり、CSVのメール列は、連絡先を正常にインポートするためにBananatagフィールド名のメールにマッピングされる必要があります。
従業員メール Webアプリについてもっと知りたいですか？詳細については、アカウントマネージャーにお問い合わせください。
