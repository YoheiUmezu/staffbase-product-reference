# Google Workspace接続のためのクライアントIDの作成

**URL**: https://support.staffbase.com/hc/ja/articles/8230423684882
**取得日**: 2026-05-02 18:46:58 JST

## 内容

Staffbaseが貴社のGoogle Workspaceに安全にアクセスできるようにするためには、Google OAuthクライアントIDとクライアントシークレットを使用してGoogle Workspace接続を構成する必要があります。
Google OAuthクライアントIDとクライアントシークレットを取得するには、以下の手順を実行してください：
プロジェクトを作成します。
Google Drive APIを有効にします。
OAuth同意画面を作成します。
クライアントIDを作成します。
前提条件
Google Enterpriseサブスクリプションを持っていること。
接続したいGoogle Workspaceアカウントの管理者権限を持っていること。
プロジェクトの作成
Google Cloud Platformにサインインします。
ダッシュボードに移動し、プロジェクトを作成をクリックします。
新しいプロジェクトを作成するページが開きます。
すでに少なくとも1つのプロジェクトを作成している場合は、ヘッダーのプロジェクトドロップダウンをクリックして、新しいプロジェクトを作成できるダイアログを開きます。
プロジェクトの名前を入力します。例：staffbase-connection
作成をクリックします。
プロジェクトが作成されました。
Google Drive APIおよびGoogle Calendar APIの有効化
各APIを個別に有効にします。各APIについて以下の手順を実行してください：
Google Drive API
Google Calendar API
Google Cloud Platformで、作成したプロジェクトを開きます。
左側のメニューから、APIとサービス > 有効なAPIとサービスに移動します。
APIとサービスを有効にするをクリックします。
Google Drive APIまたはGoogle Calendar APIを検索し、クリックします。
有効にするをクリックします。
必要に応じて、他のGoogle APIを有効にするためにこれらの手順を最初から繰り返します。
作成したプロジェクトに対してGoogle Drive APIが有効になりました。
OAuth同意画面の作成
Google Drive APIを有効にした後、StaffbaseプラットフォームからユーザーがGoogleアカウントにサインインできるようにするOAuth同意画面を構成する必要があります。ユーザーは、Staffbase従業員アプリを通じて初めてGoogleアカウントにサインインする際に、この認証画面を表示します。
Google Cloud Platformで、作成したプロジェクトに移動し、>OAuth同意画面を選択します。
ユーザータイプの下で、内部を選択します。
作成をクリックします。
アプリ情報の下で、以下を提供します：
アプリ名
ユーザーサポートメール
アプリロゴ
承認済みドメインの下で、Staffbaseプラットフォームのドメインを提供します。
開発者連絡先情報の下で、IT部門のメールアドレスを提供します。
保存して続行をクリックします。
スコープの追加または削除をクリックします。
スコープオプションを選択します：
Google Drive APIの場合：auth/drive.metadata.readonly
Google Calendar APIの場合：auth/calendar.events.readonly
更新をクリックします。
保存して続行をクリックします。
OAuth同意画面が構成され、設定の概要を確認できるようになりました。
クライアントIDの作成
プロジェクトのスコープを構成した後、クライアントIDを設定できます。クライアントIDは、StaffbaseプラットフォームでGoogle Workspace接続を構成し、貴社のGoogleアカウントをStaffbase環境に接続するために必要です。
作成したプロジェクトから、認証情報に移動します。
認証情報を作成をクリックし、ドロップダウンメニューからOAuthクライアントIDを選択します。
アプリケーションタイプの下で、ドロップダウンメニューからWebアプリケーションを選択します。
名前の下で、クライアントの名前を提供します。例：staffbase-connection-client。
承認済みJavaScriptオリジンの下で、URIを追加をクリックします。
従業員アプリまたはイントラネットにアクセスできるすべてのURIを提供します。例： https://app.staffbase.com または https://staffbase.yourdomain.com。
承認済みリダイレクトURIの下で、リダイレクトURIを提供します。
リダイレクトURIを作成するには、ドメインURIの末尾に
/api/integrations/auth/googleWorkspace/callback
最終的なリダイレクトURLの例：
https://staffbase.yourdomain.com/api/integrations/auth/googleWorkspace/callback
作成をクリックします。
クライアントIDが作成されました。このクライアントIDをコピーして、Google Workspace接続の構成に使用できます。
