# 名前でメールをパーソナライズする

アクティベーション
視聴者を引き付け、影響力を高めるために、メールをより直接的で個人的なものにしましょう。
Email Designerの名前パーソナライズ機能を使用すると、メールの本文で受信者一人一人を名前で自動的に挨拶することができます。
この機能は、Employee EmailウェブアプリまたはStudio統合から直接メールを送信する場合にのみ利用可能です。アドインや拡張機能をインストールし、メールクライアントから追跡メールを送信する場合には、この記事は該当しません。

パーソナライズオプション
Employee Emailのパーソナライズは、Microsoft Officeスイートのメールマージ機能と同様に、マージタグを使用して機能します。例えば、「hey {{firstName}},」のように、Employee Emailの連絡先ディレクトリ内の以下のフィールドを参照します：
名
姓
また、名や姓のフィールドが空の場合に置き換えるカスタムフォールバックテキストを設定することもできます。
メールを送信すると、使用している連絡先フィールドとそのフィールドが埋まっているか空であるかに応じて、受信者に異なる挨拶がされます。例えば：
「Hey {{firstName}} {{lastName}},」（フォールバックテキストが設定されていない場合）
「Hey John Doe,」名と姓のフィールドが両方とも埋まっている場合。
「Hey John,」姓のフィールドが空の場合。
「Hey Doe,」名のフィールドが空の場合。
「Hey,」名と姓のフィールドが両方とも空の場合。
「Hey {{firstName}},」
「Hey everyone,」名のフィールドが空で、フォールバックテキスト「everyone」が設定されている場合。
「Hey,」名のフィールドが空で、フォールバックテキストが設定されていない場合。

前提条件
組織がEmployee EmailウェブアプリまたはStudio統合を通じてメールを送信していること。
Email連絡先ディレクトリ内の受信者の名前が正しく入力され、メールに表示されるようにフォーマットされていること。
オプションとして、パーソナライズされたメールを送信する前に、これらのフィールドを連絡先ディレクトリで修正することができます。
Email Designerの下書きをパーソナライズする
Employee EmailウェブアプリまたはStudio統合で下書きやテンプレートを編集している間に、テキスト要素をクリックします。
テキスト編集ツールバーで、パーソナライズボタンをクリックします。
テキスト要素に{{firstName}}および/または{{lastName}}タグを追加するには、それらをクリックします。
連絡先ディレクトリで名フィールドおよび/または姓フィールドが空であり、それを置き換えるフォールバックテキストが設定されていない受信者には、メールは欠落した値やタグの前後に追加したスペースなしで送信されます。
例えば、「Hey {{firstName}},」は名フィールドが空の場合、「Hey,」として送信され、「Hey, 」とはなりません。
タグフォールバックメニューで、使用しているタグのテキストボックスに組織のデフォルト値が既に設定されているかどうかを確認します。
注: タグのフォールバックテキストを追加または変更しても、現在編集しているメールにのみ影響します。他のメール下書きやスケジュールされたメールのタグのフォールバックは更新されません。
オプションとして、特定のタグの編集ボタンをクリックして、組織のデフォルトフォールバック値を削除したり、新しいものを追加したりします。
例えば、{{firstName}}タグのフォールバックとして「everyone」と入力すると、名フィールドが空の受信者には「Hey {{firstName}},」が「Hey everyone,」として送信されます。
適用するをクリックしてフォールバックテキストを保存します。
警告: {{firstName}}と{{lastName}}タグを一緒に使用する場合、送信されたメールが次のような奇妙な挨拶を表示しないように、フォールバックがさまざまな組み合わせで意味を成すことを確認してください：
「Hey John everyone,」
「Hey everyone Doe,」

パーソナライズされたメールをプレビューする
Employee EmailウェブアプリまたはStudio統合で、パーソナライズのためのタグを含む下書きやテンプレートを編集している間に、プレビュータブをクリックします。
プレビュー情報メニューで、パーソナライズドロップダウンを開きます。
フォールバックテキストを選択して、Email連絡先ディレクトリで一部の受信者のフィールドが空の場合に表示するように設定した値でタグを置き換えます。
サンプルデータを選択して、サンプルの名および/または姓の値でタグを置き換えます。

送信前にパーソナライズ設定を確認する
パーソナライズされたメールの下書きを送信またはスケジュールする準備をしているとき、Employee EmailウェブアプリまたはStudio統合からメールを確認をクリックします。
メールを確認ページで、パーソナライズされたテキストセクションに以下が表示されます。
メールの下書きに含まれるすべてのタグ
これらのタグが参照する連絡先ディレクトリフィールドが空白の場合に設定されたフォールバックテキスト
データが欠落している受信者の数（連絡先ディレクトリの空白フィールド）
データが欠落している受信者を確認するには、詳細を表示をクリックします。
オプションとして、これらの受信者のメールアドレスをコピーし、送信前に欠落しているデータを修正します。
パーソナライズ設定に満足したら、メールを後で送信するようにスケジュールしたかどうかに応じて、今すぐ送信またはスケジュールを保存をクリックします。

送信前に名前フィールドを修正する
名前タグを含むメールを送信する際、Email連絡先ディレクトリから受信者の名前が直接メールに挿入され、元の大文字小文字、スペース、ダッシュなどに自動的な変更や検証は行われません。
時には、名または姓のフィールドが空白の場合があります。
すべての受信者がパーソナライズされたメールを受け取り、正しく宛名が付けられるように、パーソナライズされたメールを送信する前にこれらのフィールドを連絡先ディレクトリで更新できます。
最も迅速な修正方法は、CSVファイルをアップロードして、名前が欠落しているか奇妙にフォーマットされた連絡先の修正されたエントリを提供することです。
警告: 連絡先がEntra ID、Workday、またはStaffbaseアプリ/イントラネットから同期されている場合、修正されたエントリは次回同期時に上書きされる可能性があります。データをソースで検証し修正するためにITチームと協力してください。
