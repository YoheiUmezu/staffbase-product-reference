# ユーザー、コンテンツ、プロセスの統合

**URL**: https://support.staffbase.com/hc/ja/articles/360015286980
**取得日**: 2026-05-02 18:50:14 JST

## 内容

拡張機能は、ビジネスシナリオに基づいて、Staffbaseプラットフォームにさまざまな方法で統合できます。これらは2つのカテゴリに分かれます：
すぐに使える統合
組織は、日常業務で使用されるさまざまなツールやアプリケーションが連携して動作する必要があります。これらの一般的なビジネスシナリオや、人気のあるエンタープライズシステムやアプリケーションとの統合は、Staffbaseから簡単に利用できるようになっており、日常業務をよりスムーズに進めることができます。ユーザーに提供したい体験に応じて、すぐに使える統合方法のいずれかを選択してください。すぐに使える統合の下で利用可能なさまざまな統合方法は次のとおりです：
例：
ユーザーが健康当局のウェブサイトからのダッシュボードや外部アプリケーションなどの外部コンテンツに迅速にアクセスできるようにしたい場合は、それをプラットフォームにリンクできます。
ユーザーが外部コンテンツにアクセスできるだけでなく、従業員アプリやイントラネットの同じユーザーエクスペリエンスと快適さを体験できるようにしたい場合は、外部コンテンツをプラットフォームに埋め込むことができます。
ユーザーが外部ソースから自分に関連するプロセスや選択されたデータにアクセスできるようにしたい場合は、接続の方法を選択できます。ここでは、コンテンツを埋め込むことに加えて、外部ソースから特定の情報がユーザーに表示されます。これにより、プラットフォームの使用の快適さと体験が一歩進みます。
カスタムビルド統合
探している統合ソリューションがStaffbaseからすぐに利用できない場合は、特定の統合ニーズを満たすためにカスタム統合を構築するオプションがあります。これらは、非常に特定のビジネスケースを解決するために、自分自身またはベンダーや技術実装パートナーの助けを借りて構築できる統合です。カスタムビルド統合を取得したら、組織の要件に合わせてStaffbaseプラットフォームで使用できます。
カスタムビルド統合の下で利用可能なさまざまな統合方法は次のとおりです：
例：
ユーザーが外部コンテンツやプロセスに迅速にアクセスできるようにしたい場合、たとえば、休暇中の従業員のリストや社内のHRシステムからの休暇申請プロセスなどを、プラットフォームに埋め込むことができます。
ユーザーインターフェイスを提供せずに、既存のプロトコルを使用してデータを作成、更新、または削除したい場合は、APIを介して接続できます。
