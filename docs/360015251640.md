# Microsoft TeamsにおけるStaffbaseプラットフォームの統合

**URL**: https://support.staffbase.com/hc/ja/articles/360015251640
**取得日**: 2026-05-02 18:52:15 JST

## 内容

アクティベーション
管理者
アップロードしたStaffbaseプラットフォームのポリシーと設定を構成し、ユーザーの権限と許可を定義し、最後にアプリをピン留めしてユーザーが利用できるようにします。
前提条件
Microsoft 365でグローバル管理者またはTeamsサービス管理者の役割を持っている必要があります。
これから行う構成変更は、Microsoft Teamsに反映されるまでに24時間以上かかる場合があります。
ユーザー権限の構成
Microsoft Teams内でStaffbase Connectアプリにアクセスできるユーザーまたはユーザーグループを定義し、Microsoft Teamsでアプリをピン留めして簡単にアクセスできるようにします。 詳細を学ぶ。
Microsoft Teams管理センターに移動し、Teamsアプリ> セットアップポリシーに進みます。
ポリシーの管理の下で、構成したいポリシーを選択します。
ユーザーの管理をクリックします。
ユーザーを割り当てるをクリックして、Staffbaseプラットフォームにアクセスする必要があるMicrosoftユーザーを追加します。
追加したいユーザーを検索します。
名前を選択し、追加をクリックします。
適用をクリックして変更を保存します。
割り当てが有効になるまで時間がかかりますダイアログが開きます。
確認をクリックします。
アプリセットアップポリシーの構成
グローバル（組織全体のデフォルト）ポリシーを使用して、Microsoft Teamsのすべてのユーザーにアプリを利用可能にするか、カスタムポリシーを作成して選択したグループにアプリを利用可能にします。
Microsoft Teams管理センターに移動し、Teamsアプリ> セットアップポリシーに進みます。
アプリセットアップポリシー > ポリシーの管理タブで、次のいずれかを行います：
追加をクリックして、選択したグループにアプリを利用可能にするカスタムポリシーを作成します。
ポリシーを選択し、編集をクリックしてデフォルト設定を変更します。
保存をクリックします。
アプリのピン留め
追加したStaffbaseプラットフォームをアプリとしてピン留めし、Microsoft Teamsの左メニューでユーザーが簡単にアクセスできるようにします。
Microsoft Teams管理センターで、Teamsアプリ> セットアップポリシーに移動します。
更新したいポリシーを選択します。
ピン留めされたアプリの下で、アプリを追加をクリックします。
ピン留めされたアプリを追加ダイアログで、追加したStaffbaseプラットフォームを検索します。
アプリを選択します。
追加をクリックします。
アプリを選択し、上に移動または下に移動をクリックして、他のアプリの中で表示される順序を整理します。
保存をクリックします。
変更が有効になるまで時間がかかりますダイアログが開きます。
確認をクリックします。
ユーザー権限とセットアップポリシーを正常に構成し、Microsoft TeamsにStaffbase Connectアプリをピン留めしました。
追加の役立つ情報：
Staffbaseをカスタムアプリとして追加する
Microsoft Teamsでのアプリブランディングのカスタマイズ
