# アプリまたはイントラネット内のブランディング概要

**URL**: https://support.staffbase.com/hc/ja/articles/207136665
**取得日**: 2026-05-02 18:45:21 JST

## 内容

あなたの内部コミュニケーションプラットフォームは、あなたの組織の価値観や文化を表現します。Staffbase では、組織のブランドアイデンティティを内部コミュニケーションアプリに反映させるためのさまざまな方法を提供しています。これにより、従業員とのつながりを強化し、内部コミュニケーションをより強固なものにすることができます。
アプリ／イントラネット
アプリとイントラネットのブランディングページ上部の切り替えトグルを調整することで、アプリとイントラネットを切り替え、それぞれの機能を表示できます。例えば、下部のバーはアプリのみに適用され、メニューバーはイントラネットのみに適用されます。
プレビュー
アプリとイントラネットのブランディングページ上部のプレビューを使用して、ブランディングの変更がユーザーにどのように表示されるかを確認できます。トグルを切り替えて、アプリとイントラネットそれぞれの体験を確認できます。
Staffbase では、変更内容を保存する前に必ずプレビューで確認することを推奨しています。
カラー
カラーのページからブランドカラーを追加・管理できます。このページは中央リポジトリとして機能し、プラットフォーム全体でカラーを定義、カスタマイズ、コピーしやすくなります。ブランドカラーは以下のインターフェース要素に適用できます：
一部の設定は、組織が従来のメニューの外観を使用している場合は適用されません。例えば、アクセントカラーなどです。
インターフェースの色：ボタン、パンくずナビゲーション、リンクなど、主要なタスクやアクションを強調します。
ナビゲーション ヘッダー：以下のカラーオプションでナビゲーション ヘッダーの外観をカスタマイズできます：
テキストの色：ナビゲーション ヘッダー内のすべての要素（テキストやアイコンなど）に適用されます。例：メニューフォルダ、ローンチパッド、通知センター。
背景色：ナビゲーション ヘッダーの背景を設定します。
アクセント：アプリまたはイントラネットで現在開いているメニュー項目を強調表示します。
下部のバー：モバイルデバイスの下部ナビゲーションバーの色をカスタマイズして、インターフェースを強化し、ナビゲーション体験を向上させます。
メニューバー：イントラネットメニューの以下の要素をカスタマイズして、ブランドに合わせてナビゲーションを改善します。
テキストとアイコン：メニューラベルやアイコンの色を調整します。読みやすく、ブランドと視覚的に一貫性のあるスタイルを選択してください。
背景：メニューバーの背景色を設定します。コントラストが良く、全体のテーマに合った色合いを選んでください。
アクセント：有効またはホバー中のメニュー項目に使用されるハイライトカラーを定義します。ユーザーが現在の位置を把握しやすくなり、使いやすさが向上します。
Navigator：Navigator で以下の要素をカスタマイズできます：
メッセージの色：テキスト入力や応答のテキストと背景色を設定します。
ボイスオーブの色：ユーザーが音声入力を使用した際に表示されるアクセントカラーを設定します。
既存のカラーをカスタマイズしたり、HEX コードやカラーピッカーを使って追加のブランドカラーを追加できます。追加できるカラー数に制限はありません。
ロゴ
ブランディング設定で2種類のロゴをアップロードできます：
アプリとイントラネットのロゴ
組織のコアバリューを強調するロゴをアップロードします。ロゴはモバイルおよび Web アプリ全体で表示されます。正方形または長方形のロゴをアップロードできます。
要件
ロゴサイズ：
正方形ロゴ：40 x 40 px（ピクセル）
長方形ロゴ：120 x 40 px（ピクセル）
ロゴ形式：
JPG
PNG
GIF
TIF
BMP
HEIC（iOS 高効率形式）
Navigator ロゴ
Navigator 用のロゴをアップロードします。ロゴはユーザーがクリックして Navigator を開くアイコンとして表示されます。Web アプリでは右下、モバイルアプリでは下部ナビゲーションバーの上に表示されます。また、ユーザー インタラクション中も Navigator 内で表示されます。
要件
ロゴサイズ：
64 x 64 px 以上
最大 25 MB
ロゴ形式：
JPG
PNG
メインページのタイトル
アプリまたはイントラネットのホームページ用のナビゲーション ヘッダーを設定します。これは組織名、アプリ名、またはイントラネット名にすることができます。
デスクトップの背景画像
任意で、マイファイルまたはローカルコンピュータからデスクトップユーザー用の背景の画像をアップロードできます。
推奨事項：
画像サイズ 1920 x 1080 px 以上：推奨サイズと一致しない場合、画像は自動的にズームまたはトリミングされ最適なサイズに調整されますが、画質が低下する場合があります。
背景の画像は以下の場所で使用されます：
公開エリア
公開エリアがない場合はサインインダイアログの背後
サインイン後の Web アプリ全体
カスタム CSS
デフォルトのブランディングのオプションだけではブランド独自のスタイルに合わない場合、より細かいカスタマイズのためにカスタム CSS を適用できます。カスタム CSS の設定方法をご覧ください。
バナー画像付きウェルカムページ
設定されている場合、ウェルカムページはユーザーが初めてサインインした直後に表示され、アプリの紹介を行います。内部コミュニケーションプラットフォームの価値、メリット、機能を紹介したり、「ようこそ」と伝えることができます。ユーザーは初回登録後に一度だけこのページを見ます。ウェルカムページのカスタマイズは設定 > ウェルカムページで行えます。
要件：
アスペクト比 16:9
画像サイズ 1920 x 1080 px 以上
セーフゾーンに注意：画像の重要な要素が中央にあり、上下 8.5% のセーフゾーン内に収まっていることを確認してください。画像サイズが 1920 x 1080 px の場合、8.5% のセーフゾーンは高さ 91.8 px です。
