# Staffbaseメールでのカスタムデータによるパーソナライズに関するFAQ

**URL**: https://support.staffbase.com/hc/ja/articles/31210419382546
**取得日**: 2026-05-02 18:48:31 JST

## 内容

CSVファイルを作成する際に遵守すべき要件は何ですか？
最初の列には、受信者のメールアドレスまたは内部・外部IDを入力してください。この列の名前はidentifierでなければなりません。列名は大文字・小文字を区別するため、IdentifierやIDENTIFIERにはできません。ただし、メールアドレス自体は大文字・小文字を区別しません。
ファイルはUTF-8テキストエンコーディングを使用してください。
セルの区切り文字はセミコロン（;）である必要があります。
システムのパーソナライズ名（例：firstNameやlastName）をカスタムタグとして使用しないことを推奨します。代わりに、firstNameCustomなどを使用してください。
メールの下書きを複製した場合、CSVファイルを再利用できますか？
いいえ、CSVファイルは各メールごとに個別にアップロードする必要があります。メール間で保存・共有されることはありません。
アップロードしたCSVファイルを削除できますか？
はい、ゴミ箱アイコンをクリックしてCSVファイルを削除できます。その後、再度アップロードを開始できます。
CSVファイルのサンプルはありますか？
はい、CSVファイルのサンプルをダウンロードできます。ターゲット視聴者でカスタムグループを選択すると、ダウンロードオプションが表示されます。
CSVファイルに含められるパーソナライズ列の数に制限はありますか？
いいえ、パーソナライズ列の数に制限はありません。
使い捨てカスタムタグにすべてのデータ型を使用できますか？
HTMLを除き、すべてのテキスト型（リンクを含む）をパーソナライズタグとして使用できます。使い捨てカスタムタグとして含まれるリンクは、メールのアナリティクスで追跡されません。
CSVに受信者が含まれているが、その受信者の使い捨てタグ用データがない場合はどうなりますか？
メールはそのユーザーにも送信され、フォールバックテキストが使用されます。CSVアップロード後にダウンロードできるエラーレポートには、データが不足していることを示す警告が表示されます。
タグ名を手動で入力した場合、認識されたかどうかを確認するにはどうすればよいですか？
システムは大文字・小文字を区別します。正しい単語とカッコを使用すればタグは認識されます。
有効なタグは自動的に認識され、表示されます。
無効なタグ（スペルミスやCSVファイルに含まれていないタグ）は警告バナーとともに表示され、送信されません。
使い捨てカスタムタグはメールのすべての言語バージョンで利用できますか？
はい、使い捨てカスタムタグはメールの下書き全体でグローバルに利用でき、すべての言語バージョンで使用できます。
新しい言語バージョンを追加すると、既存のすべての使い捨てタグが利用可能です。
タグのフォールバックテキストは各言語バージョンごとに設定されます。新しい言語バージョンには空のフォールバックテキストフィールドが表示されます。
メールのプレビューで使い捨てカスタムタグはどのように表示されますか？
有効なタグはCSVからの実際のテキストとして表示されます。
無効なタグは削除され、表示されません。
フォールバックテキストが設定されている場合、左側のパーソナライズドロップダウンでフォールバックテキストを選択すると、タグの代わりに表示されます。
テストメールで使い捨てカスタムタグはどのように動作しますか？
テストメールがCSVファイルに含まれていないユーザーに送信された場合、タグはフォールバックテキスト、またはフォールバックテキストがない場合は何も表示されません。
テストメールがCSVファイルに含まれているユーザーに送信された場合、タグは該当するデータに置き換えられます。
