# Staffbase フィードバック管理ポリシー

**URL**: https://support.staffbase.com/hc/ja/articles/24053211779474
**取得日**: 2026-05-02 18:42:53 JST

## 内容

Staffbaseでは、お客様が私たちの最大のインスピレーションであり、プラットフォームの重要な改善を推進しています。お客様のアイデア、フィードバック、課題は、私たちのロードマップを形成するのに役立ち、真にお客様のニーズに応える製品を提供できるようにしています。
この記事では、Staffbaseがどのようにお客様のフィードバックを処理し、それを実行可能なアイデアに変換し、最終的にはプラットフォームの新機能に変えるかを説明します。
お客様は、アイデアポータルや、フィードバックオプションが利用可能なStudio内の特定の機能を通じて、Staffbaseにフィードバックを共有できます。
提出されたフィードバック
アイデアポータルを通じて提出されたフィードバックは、中央フィードバックライブラリに保存されます。Staffbaseは、自動化されたルールを使用して、類似のトピックに関するフィードバックをグループ化し、関連する製品開発チームにフィードバックをルーティングします。
グループ化されたフィードバックからアイデアへ
製品開発チームは、グループ化されたフィードバックを評価し、それをアイデアに変換します。彼らは、以下の基準に基づいてグループ化されたフィードバックを評価します：
需要: 需要は、アイデアへの投票数をカウントすることで評価されます。アイデアに関連するフィードバックの提出はすべて、1票としてカウントされます。
開発: お客様のフィードバックが、すでに検討中または開発中のアイデアや機能に関連している場合、Staffbaseはさらなる探索のためにお客様に連絡することがあります。これには以下が含まれる可能性があります：
電話またはメールのやり取り
Studioを通じたコミュニケーション
お客様成功マネージャーとの議論
これらの議論への参加は任意です。
アイデアポータルに公開されたアイデア
製品開発チームによって精査されたアイデアは、アイデアポータルに公開されます。これらのアイデアはまだ発見段階にあり、さらなる調査と検証が必要です。Staffbaseは、ロードマップにコミットする前に需要と実現可能性を評価します。
