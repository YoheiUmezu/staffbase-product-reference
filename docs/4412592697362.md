# テストメールの送信

**URL**: https://support.staffbase.com/hc/ja/articles/4412592697362
**取得日**: 2026-05-02 18:47:22 JST

## 内容

製品名に関する注意: 2022年以降、BananatagはStaffbaseのEmployee Email製品です。
テストメールは、受信者が受信トレイでメッセージをどのように表示するかを確認するための素晴らしい方法です。テストメールは追跡されず、送信されたテストメールは下書きタブから送信済みメールに移動しません。
テストメールは、WebアプリのEmail Designerから直接送信するか、メールクライアントのアドインから送信できます。Webアプリからテストメールを送信する場合、いくつかの点に留意してください:
Webアプリから送信されたテストメールの差出人アドレスはpreview@bananatag.comです。このため、メールが迷惑メールフォルダーに振り分けられたり、画像が正しく表示されない可能性があります。
画像には転送を避けるために「テストメール」の透かしが重ねられます。
テストメールは一度に1人の受信者にのみ送信できます。
Webアプリ
WebアプリのEmployee Emailで、メールの下書きまたはテンプレートを編集している間に、ダウンロードオプションアイコンをクリックします。
テスト送信を選択します。
宛先のメールアドレスを入力し、送信をクリックします。
時折、テスト受信者に画像が表示されないことがあります。これはbananatag.comからの外部送信であるため、いくつかの理由が考えられます:
Outlookの設定が外部コンテンツをブロックしている
SecureTempフォルダーが「満杯」
Wordの画像プレースホルダー設定が有効
Internet Explorer (IE)の設定で暗号化されたファイルをディスクに保存しないがチェックされている
更新情報: IE 11のサポート終了に関する情報
Outlook Classic
Outlookのリボンで、Staffbaseで作成をクリックします。
保存したEmail Designerの下書きのリストが表示されます。
テスト送信したい下書きをクリックして開きます。
テスト受信者のメールアドレスを宛先またはCCフィールドに入力します。
件名を入力します。
メールを追跡のチェックを外すか、チェックを入れたままにして、送信をクリックし、プロンプトが表示されたらテストとして送信を選択します。
Outlook 365
Employee Emailのサイドバーで新しい下書きメールを開き、下書きアイコンをクリックし、テスト送信したい下書きを見つけます。
下書きの上にカーソルを合わせて、封筒をクリックします。
追跡からテストにトグルを切り替えます。メールは現在テストモードです。
テスト受信者のメールアドレスを宛先またはCCフィールドに入力します。
件名を入力します。
テスト送信をクリックします。
Gmail
Employee Emailの拡張機能で、Email Designerの下書きのリストを開きます。
テスト送信したい下書きを見つけて送信をクリックします。
ドロップダウンフィールドから送信者アドレスを選択し、受信者を入力します。
追跡からテストにトグルを切り替えます。メールは現在テストモードです。
テスト送信をクリックします。
