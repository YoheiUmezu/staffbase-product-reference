# スペースの概要

**URL**: https://support.staffbase.com/hc/ja/articles/360010454759
**取得日**: 2026-05-02 18:44:58 JST

## 内容

スペースは、特定のユーザーに提供する必要があるコンテンツを整理・管理するためにスタジオ内に指定されたエリアです。スペースは、組織の部門に似ており、それぞれの部門の責任者によって管理され、この場合はスペース管理者によって管理されます。組織内には任意の数の部門を持つことができます。同様に、任意の数のスペースを作成することができます。部門が組織の運営により多くの構造をもたらすのと同様に、スペースはスタジオの日常管理に明確な構造をもたらします。たとえば、異なる場所にオフィスを持つ組織の場合、各オフィスのためにスペースを作成して、各場所のユーザー向けに場所特有のコンテンツを簡単に管理できます。
同様に、スペースは異なる利害関係グループ、上級管理職、新入社員のトレーニング、特別なイニシアチブ、異なる部門など、他のユースケースにも拡張できます。ユースケースは組織ごとに異なり、各組織はその構造とコミュニケーションニーズにおいて独自です。スペースは、組織のニーズに基づいて構造を作成するのに理想的で、以下のような利点を提供します：
自律的なコンテンツ作成
スペース管理者はスペースを管理し、スペース内のコンテンツに対して完全な自律性を持ち、コンテンツの作成、削除、復元、整理を行うことができます。さらに、スペース管理者は自分のスペースの分析にアクセスでき、トレンドを分析し、より効率的にスペースに対応するのに役立ちます。
明確な構造
スペースは明確に定義された構造を提供し、システム全体の管理者はプラットフォーム内のすべてのスペースの高レベルの概要を把握できます。一方、スペース管理者は自分のスペースに対してのみ明確な概要と自律性を持っています。ここで、システム管理者はスペースをスペース管理者に委任し、彼らはそれぞれのスペースに対して自律性を持ちます。また、スペース管理者はユーザー管理やシステム全体のアプリ設定にアクセスできません。これにより、上位レベルのアプリ設定にアクセスできるシステム全体の役割が多すぎることから生じる可能性のある問題を防ぎます。
簡単なコンテンツターゲティング
スペース管理者またはエディターとして、特定のスペースに移動し、コンテンツを作成、公開、管理することができ、ターゲットユーザーへの可視性を定義する必要はありません。このようにして、エディターは各個別の投稿やページで可視性を定義する必要がありません。一度設定が定義されると、それ以降に作成されるすべてのコンテンツに再利用されます。
