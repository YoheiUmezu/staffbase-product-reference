# 統合の開始

**URL**: https://support.staffbase.com/hc/ja/articles/360015287060
**取得日**: 2026-05-02 18:56:05 JST

## 内容

統合を始めることは難しい場合があります。これは、あなたが思い描いている解決策に到達する前に明確にする必要があることがいくつかあるためです。そのためには、いくつかのステップを後退させ、実際の痛点が何であるか、そして関与する各システムがそれらをどのように解決するのかを理解する必要があります。最後に、これらのシステムがあなたが思い描いている解決策にどのように接続できるかを理解する必要があります。
より良いガイダンスを提供し、目標により早く到達できるように、開始前に自分自身に問いかけるべきいくつかのヒントと質問を提供しました。
基礎作り
統合がカバーする必要があることを理解するために質問をすることで基礎作りを始めましょう。
例:
既存のアプローチにおける痛点は何ですか？
解決したいビジネスケースは何ですか？
外部システムから統合したいビジネスシナリオは何ですか？ユーザー管理、コンテンツ、またはプロセス
外部システムのデータはどこに存在しますか？クラウドベースですか、それともオンプレミスですか？
システムの性質は何ですか？一般的に使用されるエンタープライズシステムですか、それともカスタムビルドですか？
探求
これらの質問を通じて、解決策がどのように機能することを望んでいるかを思い描いてください。
例:
ユーザーエクスペリエンスはどのようであるべきですか？
外部コンテンツは元のインターフェースで表示されるべきですか、それとも異なるものにしたいですか？
どの統合方法を使用したいですか？リンク、埋め込み、または接続？
決定
ビジネスケースと解決策の両方の観点から何を望んでいるかがわかったので、統合が次のいずれかであるかを確認してください：
すぐに利用可能: その場合は、すぐに使用できる統合を使用してください。
すぐに利用可能でない: その場合は、統合の目的のためにそれを構築する必要があります。カスタムビルドの統合について詳しく学び、その後、Staffbaseサポートまたはカスタマーサクセスマネージャーに相談してください。
