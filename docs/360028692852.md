# プラットフォームにおけるコンテンツの寿命

**URL**: https://support.staffbase.com/hc/ja/articles/360028692852
**取得日**: 2026-05-02 18:45:07 JST

## 内容

Staffbaseプラットフォームは、アプリユーザーのレーダー上で異なる寿命を持つコンテンツを収容するように設計されています。このような設計は、コンテンツのエンゲージメントと共有を最大化し、コミュニケーションの改善をもたらします。
寿命に応じたさまざまなタイプのコンテンツを見てみましょう：
短寿命コンテンツ
このコンテンツは、ユーザーのレーダー上で短期間アクティブであり、非アクティブになるか、完全に関連性を失ったために消える可能性があります。コンテンツの短い寿命は、他のコンテンツと比較して関連性が低いことを意味するわけではなく、関連性が限られているだけです。この短い寿命の間に、最近のイベントに関するコメントなど、高いレベルのソーシャルインタラクションを生み出すことが多く、このコンテンツは非常にダイナミックです。最新のコンテンツは、ニュースフィード内の古いコンテンツを置き換えます。これは、新しいコンテンツが公開される頻度にも依存します。
通常、短寿命のコンテンツは、アプリ内で更新投稿やメディア投稿として公開されます。このようなコンテンツには、駐車スペースの非利用、従業員の歓迎、スケジュール変更のお知らせ、イベントの発表、今日の写真などがあります。
ダイナミックでインタラクティブなコンテンツを設定するには、ニュースクイックスタートガイドを参照してください。
中寿命コンテンツ
このコンテンツは、ユーザーのレーダー上でしばらくアクティブであり、非アクティブになるか、関連性を失ったために消える可能性があります。時間が経つにつれて一部のコンテンツが関連性を失うことがありますが、未公開または削除されない限り、検索を通じてコンテンツを見つけることができます。
通常、中寿命でダイナミックなコンテンツは、アプリ内で記事投稿や時にはメディア投稿または更新投稿として公開されます。中寿命コンテンツは定期的に古くなり、新しいコンテンツに置き換えられることが多いです。このようなコンテンツには、Q4のマーケティング戦略、生産性の向上、月間写真コンテスト、掲示板などがあります。
中寿命コンテンツを定期的に公開するには、ニュースクイックスタートガイドを参照してください。
長寿命コンテンツ
このコンテンツは、高い関連性と永続的な性質、または古くなる可能性が限られているため、ユーザーのレーダー上で長期間アクティブです。
通常、比較的静的なコンテンツは、アプリ内でページとして公開されます。静的ページはニュースフィードには表示されません。ページは、会社について、問い合わせ、ミッションとビジョン、安全ガイドラインなどを通知するのに最適な場所です。
静的ページを設定するには、ページクイックスタートガイドに従ってください。
