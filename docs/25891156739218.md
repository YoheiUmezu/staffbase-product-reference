# ブロッキング承認の概要

**URL**: https://support.staffbase.com/hc/ja/articles/25891156739218
**取得日**: 2026-05-02 19:02:52 JST

## 内容

公開された投稿を管理することは、コンプライアンスや品質保証のために重要です。Blocking Approvals（ブロッキング承認）は、指定された承認者から明確な承認がない限り、コンテンツを公開できないようにする構造化されたワークフローを提供します。このプロセスにより、組織は基準を徹底し、無断公開を防ぎ、コンテンツの一貫性を維持できます。
有効にすると、Studioにアクセスできるコンテンツ作成者は、投稿の承認をリクエストする必要があります。承認者には、メールで投稿へのリンクが送信されます。承認者がStudioへのフルアクセス権を持っているかどうかに応じて、Studio内の投稿ページまたはプラットフォーム上の閲覧専用ページに移動します。
Blocking Approvals（ブロッキング承認）はチャンネルごとに設定されます。特定のチャンネルでBlocking Approvalsが有効になると、そのチャンネル内のすべての投稿は公開前に承認が必要です。
Blocking Approvalsのメリット
コンプライアンスの確保とリスクの低減
明確な承認なしにコンテンツが公開されるのを防ぎ、すべての公開資料がコンプライアンス要件や社内ガイドラインを満たしていることを保証します。これにより、エラーや誤情報、潜在的なリスクを減らすことができます。
承認プロセスの効率化と一元化
コンテンツの承認フローを明確に構築し、承認やフィードバックをプラットフォーム内で一元管理することで、手動でのやり取りを減らします。
透明性とアクセス性の向上
承認状況、期限、割り当てられた承認者を一目で確認できます。法務や財務など、Studioへのフルアクセス権がない承認者も、投稿の承認や却下が可能です。
承認ステータスの説明
承認ステータスは、承認申請された投稿の現在の状態を把握するのに役立ちます。各ステータスは、申請者または承認者の操作に応じて自動的に割り当てられます。
