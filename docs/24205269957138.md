# Staffbaseメールにおけるユーザーの権限と役割の概要

**URL**: https://support.staffbase.com/hc/ja/articles/24205269957138
**取得日**: 2026-05-02 18:46:20 JST

## 内容

StaffbaseメールをStaffbaseイントラネットまたは従業員アプリと一緒に使用している場合は、権限と役割をご参照ください。以下の情報は、Staffbaseメールを単体製品として使用する場合のみに適用されます。
Staffbaseメールを効果的に利用するには、まずユーザーベースを正しく設定する必要があります。Staffbaseメールのユーザーベースは、2種類のユーザーで構成されています：
コンタクト：メールを受信することのみ可能で、Studioにアクセスできないユーザー。
Studioユーザー：Studioにアクセスでき、権限に応じたアクションを実行できるユーザー。
Studioユーザーも、ターゲットメール用のユーザーグループに所属していれば、コンタクトと同様にメールを受信できます。
ユーザー中心の役割
Studioユーザーのみに役割を割り当てることができます。役割を付与することで、ユーザーは付与された権限に基づいて特定のタスクを実行できます。
Staffbaseメールにおけるユーザー中心の役割は以下の通りです：
管理者
最も強力な役割であり、Studio内のすべてのアクションを実行できます。すべてのタスクを実行できますが、主なタスクはプラットフォーム上のユーザー管理です。
編集責任者
このStudio役割は、ユーザー管理と設定を除き、管理者が利用できるすべてのアクションにフルアクセスできます。
ユーザー
Studioへのアクセスが制限された役割です。管理者は、Studio内で特定のタスクを実行するための追加権限を付与できます。
Studioユーザー向けのコンテンツ中心の役割
管理者は、ユーザーに以下の追加権限を割り当てることができます：
フォルダ権限
管理者と編集責任者は、すべてのフォルダを削除する権限を持っています。スペース管理者は自分のスペース内のフォルダを削除できます。
フォルダを作成する際に、次のいずれかの権限を付与できます：
フォルダ管理者：メール内の特定のフォルダに対して管理タスクを実行できます。
フォルダ編集者：メール内の特定のフォルダに対して編集タスクを実行できます。

スペース権限
プラットフォームでスペースが利用可能な場合、ユーザーにスペース管理者権限を割り当てることができ、指定されたスペースのすべての管理が可能になります。
テンプレートとギャラリーの権限
テンプレート管理者を割り当てることで、テンプレートやギャラリーの変更ができるユーザーを制御できます。テンプレート設定でテンプレート管理者の役割をユーザーやユーザーグループに付与できるのは、管理者、編集責任者、または「全従業員」スペースのスペース管理者のみです。以下の表は、各ユーザー役割のテンプレート権限を示しています。

Studioユーザーのステータス
Studioユーザーは、プラットフォームに招待された後、次の3つの段階があります：
保留中：招待されたが、まだ登録手続きを完了していないStudioユーザー。
登録済み：Studioにアクセスでき、役割に応じたアクションを実行できるStudioユーザー。
無効化済み：Studioへのアクセスが許可されていないStudioユーザー（アクセスが再度許可されない限り）。
