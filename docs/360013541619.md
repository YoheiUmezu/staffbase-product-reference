# SCIM 2.0によるユーザー管理

**URL**: https://support.staffbase.com/hc/ja/articles/360013541619
**取得日**: 2026-05-02 18:54:22 JST

## 内容

アクティベーション
管理者
Staffbaseは、クロスドメインアイデンティティ管理システム（SCIM）を使用したユーザー管理をサポートしています。SCIMを使用すると、Staffbaseプラットフォーム上でアイデンティティプロバイダーを利用して、以下の方法でユーザーを管理できます：
組織に参加した際にプラットフォームにユーザーをオンボードする
関連するグループにユーザーを追加するためのユーザーグループを作成する
組織を離れた際にプラットフォームからユーザーをオフボードする
もはや必要のないユーザーグループを削除する
SCIMは、ユーザー管理が安全で便利かつ標準化されていることを保証します。私たちは、以下のいずれかのアイデンティティ管理システムを通じて、顧客がSCIMを利用できるように成功裏に支援してきました：
Microsoft Entra ID（旧Azure Active Directory）
Okta
One Identity Manager
SailPoint
異なるアイデンティティ管理システムを使用している場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーに連絡して、ユーザー管理を簡素化するためにSCIMを使用する方法を学んでください。
StaffbaseはSCIMプロトコルの主要な側面をサポートしています。ただし、ソースシステムの特定のSCIM実装や、現在Staffbase内で利用可能な機能によっては、特定の要素が完全にサポートされていない場合があります。
SCIMを使用する利点
SCIMは、CSVアップロードやユーザーAPIなどの従来の自動ユーザー管理に対して、さまざまな利点を提供します。いくつかの利点は次のとおりです：
スケーラブルなモデル：組織が使用するさまざまなツールやアプリケーション全体でのユーザー管理に最適で、ユーザーライフサイクルを中央のポイントから管理できることを保証します。
安全性：自動的かつ中央集権的なユーザー管理により、組織内の複数のアプリケーションで古くなったユーザープロファイルのリスクが低減されます。たとえば、従業員が雇用期間の終了時に組織を離れるとき、SCIMを使用することで、ユーザープロファイルが適時に更新されます。これにより、これらのユーザープロファイルへのすべてのアプリケーションへのアクセスが迅速に取り消されます。
コストと複雑さの削減：既存の標準に基づいて構築され、ITチームがすべてのユーザープロファイルを最新の状態に保ち、組織内で使用されるすべてのツールやアプリケーションと同期させるための労力を削減します。
ユーザー管理のためにSCIMを設定する方法について詳しく知りたい場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
