# CSVによる条件付きグループ管理の例

**URL**: https://support.staffbase.com/hc/ja/articles/360010648300
**取得日**: 2026-05-02 18:54:23 JST

## 内容

CSVを使用した条件付きグループの管理は、大規模なグループ管理において時間効率の良いオプションです。グループの基準となるタグや内部グループを設定したら、ユーザーは簡単なCSV同期で任意の条件付きグループに追加または削除できます。
オフィスの所在地と部門のプロファイルフィールドにタグ付けを有効にし、対応する条件付きグループを作成したとします。これらの条件付きグループにユーザーを追加および削除するにはどうすればよいでしょうか？
個々のユーザーに対するすべての変更は、ユーザーCSVで行われます。
条件付きグループに新しいユーザーを追加する
新しい同僚が、あなたの会社のドレスデンオフィスのマーケティング部門に参加しました。この新しい同僚をマーケティング部門およびドレスデンオフィスの条件付きグループに追加するために、マーケティングおよびドレスデンというタグが、これらのタグ用に指定された列に配置されました。
条件付きグループからユーザーを削除する
以前ドレスデンオフィスで働いていたマーケティングの同僚が転居し、現在はリモートで働いています。このユーザーをタグベースのドレスデンオフィスグループから削除するには、ドレスデンというタグを所在地タグ用に指定された列から単に削除する必要があります。このタグを削除することで、この同僚は所在地ベースの条件付きグループから削除されますが、マーケティング部門の条件付きグループには残ります。条件付きグループ用に指定された列のセルを空にすると、対応するグループからユーザーが削除されます。同様に、同僚が組織を退職した場合、ユーザーの行を削除することで、そのユーザーはすべての条件付きグループから削除されます。
この例を使用して、これらの方法をより広範囲に適用し、CSV内の任意のプロファイルフィールドやユーザーに対して変更を行うことができます。CSV同期を実行すると、すべての変更が同時に従業員アプリに更新されます。これにより、条件付きユーザーグループは、ユーザーCSVで行われた変更を反映するように自動的に更新されます。
