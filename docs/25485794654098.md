# プラットフォームのメインメニューオプション

**URL**: https://support.staffbase.com/hc/ja/articles/25485794654098
**取得日**: 2026-05-02 18:42:19 JST

## 内容

メインメニューはアプリの設計図です。メニュー内のコンテンツの整理方法は、ユーザーがプラットフォームを体験し、相互作用する方法にとって重要です。各ユーザーはプラットフォームを使用する際に異なるニーズを持っているため、完璧なメニュー構造は存在しません。しかし、多くのユーザーに合わせてアプリをカスタマイズする方法はたくさんあります。直感的な階層と明確なラベリングは、プラットフォームをできるだけ多くのユーザーにとって使いやすく、アクセスしやすくするための最も重要な側面の2つです。
Staffbaseプラットフォームでは、次のようなさまざまなメニューのカスタマイズが可能です：
異なるメニューレイアウト
メニュー項目のカスタマイズ可能なタイトルとアイコン
デバイスタイプに基づくコンテンツの可視性
ユーザーグループやスペースに基づくコンテンツの可視性
Webアプリのメニュー構造
Webアプリはメニュー項目のための十分なスペースを提供します。メニュー項目は、ページ、ニュースチャンネル、プラグイン、フォルダーで構成され、メニュー内に直接表示されるアイコンとペアになります。コンテンツをどのように整理したいかに応じて、Staffbaseはプラットフォームのメインメニューを表示するための2つのオプションを提供します：
標準縦型メニュー
このメニューはWebアプリのインターフェースの側面に縦に表示されます。メニューパスは最大5レベルの深さを持つように構成できます。このメニュー構造は、従業員アプリをニュースおよびコミュニケーションチャネルとして使用する組織など、さまざまなビジネスケースに適しています。
縦型メニューはエレガントなソリューションであり、異なるユーザーに対してメニューがどのように見えるかを視覚化する管理者の作業を簡素化します。高度な可視性設定により、最終的な出力は異なるユーザーや異なるデバイスタイプで異なる見え方になりますが、スタジオとユーザーインターフェース間の類似した構造は管理者に多くのガイダンスを提供します。同様に、Webおよびモバイルアプリインターフェースの平行構造により、ユーザーはWebからモバイルアプリに切り替える際に簡単にナビゲートできます。
横型メニュー
アクティベーション
横型メニューは、トップナビゲーションとも呼ばれ、ページの上部に横に表示され、最大3層のフライアウトメニューを許可します。フライアウトは、ユーザーが多数のフォルダーをクリックすることなく、長いメニューパスを表示できるため、便利です。横型メニューは、Staffbase Webアプリをイントラネットソリューションとして使用したり、HRプロセスやサードパーティツールへのアクセスなどのサービスを統合したりすることを望む組織に最適です。
Webアプリでの横型出力にもかかわらず、スタジオおよびモバイルアプリでのメニュー構造は縦型形式で表示されます。これにより、異なるデバイスタイプのユーザーに対してメニューがどのように見えるかを視覚化するのが非常に複雑になる可能性がありますが、異なるメニュー構造は、ユーザーが使用しているデバイスに応じて最適なアプリ体験を提供するのに役立ちます。
フォルダー画像を選択する際は、フォルダー画像の推奨サイズが1000px x 400px、フライアウトメニューのサイズが230px x 115pxであることを考慮してください。
モバイルアプリのメニュー構造
Webアプリの縦型または横型メニューを選択したかどうかにかかわらず、モバイルアプリのメニューは常にモバイル画面の側面に縦型形式で開きます。モバイルデバイスのメニュー空間は、ラップトップやデスクトップ画面よりもはるかに限られているため、モバイルアプリメニューのメニュー項目は、モバイルユーザーの異なるニーズに応じてより慎重に選択する必要があります。
これは、デバイスタイプに基づく可視性設定を使用することで可能です。各メニュー項目について、どのデバイスタイプで表示するかを選択します。モバイルオプションの選択を解除して、モバイルメニューの整理を開始できます。
モバイルメニューの整理
組織内のモバイルユーザーにとって重要な機能を特定し、モバイルメニューで優先順位を付けます。モバイルユーザーが必要とする情報の種類について考えてみてください。例えば：
ユーザーはアプリでシフトプランにアクセスする必要がありますか？
外出先で連絡先情報のアクセスが必要ですか？
または、チャットで同僚から迅速なフィードバックを受け取る能力が必要ですか？
モバイルメニューをカスタマイズするための別のヒントは、デスクトップアプリケーションへのアクセスを削除することです。たとえば、モバイルデバイスでサポートされていないプログラムへのリンクや、モバイルデバイスでアクセスが難しいタスクやドキュメントへのリンクを削除することで、ナビゲーションが大幅に簡素化されます。
再設計されたモバイルメニューの使用
ベータ
アクティベーション
前提条件
最新のモバイルアプリバージョンを使用しています。
モバイルデバイス用のクイックリンクナビゲーションを有効にしています。
モバイルデバイスのナビゲーションバーは、現代的な外観と感触を持っています。メニューアイコンは下部ナビゲーションバーにあり、5番目のクイックリンクの位置を占めています。再設計されたナビゲーションバーでは、メニュー用に5番目のリンクが使用されるため、4つのクイックリンクのみを追加できます。
メニュー項目をCSVとしてエクスポートして、ナビゲーションバーのクイックリンクを特定し、調整できます。
ナビゲーションショートカット
メニューは単独では存在しません。Staffbaseプラットフォームは、メニューをサポートし、ショートカットとして機能するさまざまな他の機能を提供します。これらの追加機能のいずれかをメニューに補完することで、特定のユーザーグループにターゲットを絞ることができ、コンテンツを配布し、メニューのコンテンツ負荷を軽減し、ユーザーが頻繁に使用するツールに迅速にアクセスできるようにすることができます。
ウィジェット
ウィジェット（リンクリストウィジェット、リンクタイルウィジェット、ポートフォリオウィジェットなど）をスタートページや特定のランディングページで使用することで、メニューが過負荷になる可能性のあるコンテンツを分散させることができます。これらは、魅力的でナビゲートしやすい方法で、さまざまなリンクやコンテンツへの迅速なアクセスを提供するのに最適です。また、最も使用されるクイックリンクを表示したり、部門のトピックなどのカテゴリをまとめたりするためにも使用できます。
ランチパッド
ランチパッドは、すべてのサードパーティアプリをまとめるための優れたツールです。外部ツールへのすべてのリンクをランチパッドに配置することで、ユーザーはランチパッドを従業員アプリから別のアプリケーションに飛び出すための踏み台として認識します。ランチパッドアプリは各従業員の使用状況に基づいて整理されており、特に個人的なタッチを与えます。高度にカスタマイズ可能なランチパッドは、モバイルおよびウェブアプリに合わせて調整できるため、ユーザーは使用しているデバイスタイプに利用可能なアプリのみを表示します。
モバイルクイックリンクナビゲーション
モバイルデバイス用のクイックリンクナビゲーションバーは画面の下部に表示され、最大5つのリンクを定義できます。これにより、ユーザーはアプリ内の頻繁に使用される機能にワンクリックでアクセスできます。
再設計されたモバイルナビゲーションを使用する際、メニューアイコンは下部ナビゲーションバーに配置され、5番目のクイックリンクの位置を占めます。このため、再設計されたナビゲーションバーでは、4つのクイックリンクしか表示できません。
始めるためのベストプラクティス
最適なナビゲーション構造は組織によって異なりますが、いくつかのベストプラクティスが正しい道を歩む手助けになります。
トップレベルのナビゲーションには8つを超えないトピックを選択してください。
業務関連のコンテンツとソーシャルコンテンツを分けてください。
コンテンツの重複を減らし、ユーザーが情報を見つけるためにどのメニュー項目の下にあるかを推測する必要がないようにします。
実装前にワークショップやユーザーテストでメニュー構造をテストしてください。
従業員アプリが組織と共に進化するように、メニュー構造を継続的にテストしてください。
最適なメニュー構造を見つけるためのより具体的なガイダンスについては、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
