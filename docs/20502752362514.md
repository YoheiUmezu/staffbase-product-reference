# StaffbaseへのAtlassian接続の追加

**URL**: https://support.staffbase.com/hc/ja/articles/20502752362514
**取得日**: 2026-05-02 18:49:02 JST

## 内容

ベータ
アクティベーション
管理者
ConfluenceとStaffbaseの統合により、プラットフォーム間でコンテンツを見つけるのがこれまで以上に簡単になります。このFront Door Intranet用の統合を使用すると、Staffbaseプラットフォームに存在するコンテンツだけでなく、Confluenceのコンテンツにもアクセスできます。
Confluence接続を追加して、ユーザーがStaffbaseプラットフォームからConfluenceコンテンツを検索してアクセスできるようにします。
前提条件
あなたまたはAtlassian管理者が、Atlassian Developer Consoleで以下の情報を生成しています：
StaffbaseがAtlassianインスタンスにアクセスするためのエンドポイント
クライアントID
クライアントシークレット
Studioで、設定 > サードパーティ接続に移動します。
Atlassianの下で、追加をクリックします。
Atlassianへの接続を追加するためのダイアログが開きます。
Atlassianで生成した情報を使用して、以下のフィールドに入力します。
- インスタンス名
- クライアントID
- クライアントシークレット
インスタンス名は、atlassian.netの前にある名前です。atlassian.netの前に名前がない場合は、「app」をインスタンス名として使用します。
ドロップダウンメニューからAtlassian統合の可視性を選択します。
追加をクリックします。
AtlassianをStaffbaseに正常に接続しました。この接続では、まだConfluenceコンテンツにアクセスできません。ユーザーが利用できるようにするには、StudioでConfluence検索を有効にする必要があります。
StaffbaseでのConfluence検索の有効化
ユーザーが利用できるようにするために、StudioでConfluence検索を有効にする必要があります。
Studioで、設定 > サードパーティ接続に移動します。
Atlassianの下で、展開アイコンをクリックします。
Confluence検索のトグルを有効にします。
Confluence検索を有効にするためのダイアログが開きます。
アクティブ化をクリックします。
StaffbaseプラットフォームでConfluence検索を正常に有効にしました。これ以降、ユーザーがグローバル検索を使用すると、Confluence用の専用タブが表示されます。Confluenceタブの下で、ユーザーがAtlassianアカウントにサインインしている場合、特定の検索用語に対するすべての検索結果がリストされます。
Confluence検索統合を有効にした後、初めてグローバル検索を使用する際には、ユーザーがAtlassianの資格情報でサインインする必要があります。
