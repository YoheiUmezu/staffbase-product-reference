# CSVによる必須グループ管理の例

**URL**: https://support.staffbase.com/hc/ja/articles/360010687180
**取得日**: 2026-05-02 18:50:15 JST

## 内容

オープングループのグループ識別子を設定したら、ユーザーを必須メンバーとしてオープングループに追加し、簡単なCSV同期で削除できます。
オフィスの場所間での透明性とエンゲージメントを促進したいと考え、すべての従業員がアクセスできるようにオフィスの場所や部門に基づいてグループを作成することに決めたとします。特定の部門の従業員や特定の場所に基づく従業員が重要なコンテンツから退会しないようにするにはどうすればよいでしょうか？
個々のユーザーに対するすべての変更はユーザーCSVで行われます。
必須グループでユーザーを必須として設定する
新しい同僚が、あなたの会社のドレスデンオフィスのマーケティング部門に参加しました。この新しい同僚をマーケティング部門およびドレスデンオフィスグループの必須オープングループに必須メンバーとして追加するために、識別子 Marketing_idとDresden_idがオープングループ識別子用に指定された列に配置されています。
必須グループからユーザーを削除する
マーケティング部門の同僚がドレスデンオフィスを離れ、現在はリモートで働いています。この同僚をドレスデンの必須グループから削除するには、必須グループ識別子用に指定された列からDresden_id識別子を削除します。このタグを削除するだけで、この同僚は場所に基づく必須グループからのみ削除され、マーケティングの必須グループには残ります。このユーザーは、他の非必須ユーザーと同様に、ドレスデングループのコンテンツに自由に登録および退会できるようになります。必須グループ用に指定された列のセルを空にすると、ユーザーはすべての必須グループから削除されます。同様に、同僚が組織を離れた場合、ユーザーの行を削除することで、そのユーザーはユーザーグループから削除されます。
これらの例を使用して、これらの方法をより広範囲に適用し、CSV内の任意の数のユーザーに変更を加えることができます。CSV同期を実行すると、すべての変更が従業員アプリに同時に更新されます。これは、必須ユーザーグループがユーザーCSVで行われた変更を反映するように自動的に更新されることを意味します。
