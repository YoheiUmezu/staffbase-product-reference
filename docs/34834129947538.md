# ページ移行に関するよくある質問（FAQ）

**URL**: https://support.staffbase.com/hc/ja/articles/34834129947538
**取得日**: 2026-05-02 19:02:02 JST

## 内容

ベータ
アクティベーション
円滑な移行のためにできることは何ですか？
リスクの低いページから始め、ページ所有者を早期に関与させ、移行済みページを公開前にレビューしてください。
大規模な組織の場合は、段階的な移行を使用します。クリティカルなページを準備し、カスタマイズを確認し、編集者のトレーニングを行ってください。その後、影響の大きいコンテンツを優先して徐々に移行します。
移行期間中はクラシックエディターとモダンエディターが共存します。更なる情報は、カスタマーサクセスマネージャーまたはStaffbaseサポートにお問い合わせください。
移行はAIによって行われますか？
いいえ。移行は決定論的なルールベースの方法で行われます。
すべてのページを移行できますか？
最初はできません。サポートされていないブロックや高度なカスタマイズ、複雑なネスト構造を使用しているページは、初期段階では対象外となる場合があります。
準備完了とマークされたページのみ移行できます。
どのページが移行準備完了か確認できますか？
はい。Content Designer準備完了タブに、移行可能な全ページがリストされます。
非常に長いページや複雑なページはどうなりますか？
複雑なページも移行可能ですが、構造によっては対象外となる場合があります。深いネストやレガシーウィジェットは準備状況に影響します。
移行は実際のエンタープライズページで検証され、複雑なケースもベータ検証プロセスに含まれています。
ページが移行された後はどうなりますか？
確定後、移行済みページがクラシックバージョンを置き換えます。重複ページは作成されません。
クラシックページとモダンページは共存しますか？
はい。移行期間中は両方のエディターが共存します。移行はページごとに行われ、初期段階ではクラシックエディターも利用可能です。
移行を強制されますか？
いいえ。ベータ期間中は移行は必須ではありません。完全な移行スケジュールは後日決定されます。
CSSや高度なカスタマイズはどうなりますか？
カスタマイズの互換性は現在確認中です。長期的なシステム安定性のため、一部のCSSの柔軟性が制限される場合があります。
マルチブランディングなどのエンタープライズ設定は早期に確認してください。Staffbaseは、移行計画前に高度にカスタマイズされた環境を特定することを推奨します。
アナリティクスやデータは保持されますか？
はい。ページのステータス、表示、アナリティクスの継続性は移行中も維持されます。
