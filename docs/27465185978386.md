# 機能ステータスの概要

**URL**: https://support.staffbase.com/hc/ja/articles/27465185978386
**取得日**: 2026-05-02 18:48:36 JST

## 内容

スタッフベースには、製品ライフサイクルに基づいたさまざまな種類の機能ステータスがあります。各機能に適したステータスは、製品チームと関係者によって決定されます。これらのステータスは、機能がどの開発段階にあるか、またどのようにお客様に提供されるかを明確にするのに役立ちます。
機能は、製品ライフサイクルのさまざまなフェーズを異なるスピードで進みます。最初は社内テストが行われ、その後クローズドベータとして、限られたお客様グループで一定期間テストされます。この最初のテスト段階が完了すると、機能は次のいずれかの機能ステータスで提供されます：
ベータ
プロダクション
新しい機能や更新、そのステータスについては、スタッフベースのリリースプロセスを通じて常に通知されます。ベータおよびプロダクションの両方の機能は、サポートポータルのリリースノート、マーケティングコミュニケーション、サポートおよび開発者ポータルのドキュメントで公式に案内されます。
ベータ
ベータ機能は、まだ開発中ですが、テストできる安定した状態です。ベータの主な目的は、不具合や使い勝手の問題を特定し、プロダクションに移行する前に機能が品質基準を満たしていることを確認することです。最終バージョンの形作りに重要な役割を果たすため、フィードバックの共有も強く推奨されています。
すべてのマーケティングおよびサポートドキュメントには、リリースノートのベータラベルやマーケティング資料での明記など、機能がベータであることが明確に示されています。これは、機能が最終テスト段階にあることを明確にするためです。ベータはオプトイン方式で、必ず有効化が必要です。また、一部の機能は特定の料金プランでのみ利用可能な場合があります。ベータ機能の料金情報はサポートポータルのリリースノートには記載されていませんが、詳細はカスタマーサクセスマネージャーまたはスタッフベースサポートにお問い合わせいただけます。
プロダクション
プロダクション機能は、完全に開発され、お客様に正式にリリースされたものです。これらは製品の一部として提供され、利用促進を目的に設計されており、積極的に保守・改善が行われます。スタッフベースは、お客様からのフィードバック、パフォーマンスのインサイト、進化する製品目標に基づき、プロダクション機能を継続的に強化しています。
機能によっては、有効化、段階的展開、自動展開のいずれかで提供されます。展開方法については、リリースプロセスを通じて常に通知されます。これらは製品のエキシビットおよび料金プランに含まれています。プロダクション機能の料金は、サポートポータルのリリースノートに記載されています。
