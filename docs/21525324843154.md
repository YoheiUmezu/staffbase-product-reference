# スタジオでのニュース投稿の高度なコンテンツレビュー機能の使用

**URL**: https://support.staffbase.com/hc/ja/articles/21525324843154
**取得日**: 2026-05-02 18:51:20 JST

## 内容

アクティベーション
エディター
従業員を引き込む魅力的なニュースを作成するには、時間、労力、チームワークが必要です。ニュース投稿のコンテンツレビュー機能を使用すると、アイデアを簡単に議論したり、フィードバックを求めたり、承認を得たりできます。これはすべてのニュースチャンネルタイプで利用可能です。
投稿作成者は、ニュース投稿を共有することでコラボレーターを招待できます。招待されたコラボレーターは、メールを通じてアクセスを受け取り、その後、スタジオ内のレビュータブでニュース投稿にコメントを残すことができます。
高度なコンテンツレビューを使用すると、次のことができます：
コメントを追加し、他の人のコメントを見る
コメントを編集
コメントを削除
ユーザーを@でメンションする
ユーザーにタスクを割り当てる
ユーザーに承認をリクエストする
新しいコメントやメンションに関するメール通知を受け取る
レビュータブでのニュース投稿へのコメント
レビュータブでは、新しいコメントを追加したり、既存のコメントに返信したりすることで、ニュース投稿にフィードバックを残すことができます。これにより、明確で構造化されたコメント履歴が確保されます。
コメントの追加
スタジオで、コメントしたいニュース投稿に移動します。
レビュータブに移動します。
コメントボックス内をクリックしてフィードバックを書きます。
コメントをクリックします。
ニュース投稿のレビュータブにコメントを追加しました。作成者と他のすべてのレビュアーがあなたのコメントを見ることができます。投稿作成者は、新しいコメントに関するメール通知を受け取ります。
スレッド内のコメントへの返信
初期コメントがすでに存在する場合にのみ、コメントに返信できます。スレッド内のコメントでは、同僚を@メンションできますが、タスクの割り当てや承認リクエスト機能は使用できません。
スタジオで、コメントしたいニュース投稿に移動します。
レビュータブに移動します。
返信したいコメントに移動します。
返信をクリックします。
コメントボックス内をクリックしてフィードバックを書きます。
返信をクリックします。
返信はコメントの下にスレッドとして追加されます。
ニュース投稿のレビュータブにコメントへの返信を追加しました。作成者と他のすべてのレビュアーがあなたのコメントを見ることができます。投稿作成者は、新しいコメントに関するメール通知を受け取ります。
レビュータブでのニュース投稿のコメントの編集
レビュータブでは、スペルミスを修正したり、フィードバックを更新したり、考えを洗練させたりするために、提出したコメントを編集できます。これにより、クリーンで一貫したレビュースレッドが確保されます。
自分のコメントは、返信がまだない場合にのみ編集できます。コメントに返信があった場合、そのコメントはもはや編集できません。スレッド内では、最も最近追加されたコメントのみが編集可能です。
スタジオで、コメントを編集したいニュース投稿に移動します。
レビュータブに移動します。
編集したいコメントに移動します。
三つの点をクリックします。
編集をクリックします。
コメントボックスが編集モードで開きます。
コメントを編集します。
保存をクリックします。
ニュース投稿のレビュータブでコメントを編集しました。コメントは編集済みと表示されます。
レビュータブでのニュース投稿のコメントの削除
レビュータブでは、スレッド全体または単一の返信を削除できます。
自分のコメントのみを削除できます。
スタジオで、コメントを削除したいニュース投稿に移動します。
レビュータブに移動します。
削除したいコメントに移動します。
三つの点をクリックします。
スレッドを削除または返信を削除をクリックします。
ダイアログが開きます。
削除をクリックします。
ニュース投稿のレビュータブでスレッドまたは返信を永久に削除しました。
コメントでの@メンションの使用
この@mentions機能を使用すると、ニュース投稿のレビュータブで同僚をタグ付けできます。メンションされたユーザーはメールを受け取り、レビュープロセスをできるだけ迅速かつ効率的に進めることができます。
前提条件
メンションされたユーザーが、個別またはユーザーグループの一部として、スペースの可視性フィールド選択に含まれていること。
メンションされたユーザーが、個別またはユーザーグループの一部として、チャンネルのアプリ内可視性フィールド選択に含まれていること。
メンションされたユーザーが、共有機能を使用してニュース投稿に招待されていること。
可視性要件が満たされていない場合、@の後に名前を入力してもドロップダウンメニューにユーザーが表示されません。これには、エディター、マネージングエディター、管理者などのシステム全体の権限を持つユーザーが含まれます。可視性要件が満たされているが、ユーザーがニュース投稿に招待されていない場合、@メンションに関するメールが送信されますが、メール内のリンクをクリックするとエラーページが表示されます。
スタジオで、同僚をメンションしたいニュース投稿に移動します。
レビュータブに移動します。
コメントフィールドに@の後にユーザー名を入力します。
適切なユーザーのリストが表示されます。
メンションしたいユーザーを選択します。
オプションで、コメントフィールドにさらに指示を書きます。
コメントをクリックします。
コメントがコメントフィールドの下に表示されます。
ニュース投稿のレビュータブでコメントにユーザーをメンションしました。ユーザーはメンションに関するメールを受け取ります。メールにはニュース投稿へのリンクが含まれています。
コメントでのタスクの割り当て
レビュータブでのコラボレーターへのタスクの割り当ては、ニュース投稿作成中の最良のコラボレーションを確保します。
未完了のタスクは、投稿の公開を妨げません。
スタジオで、タスクを割り当てたいニュース投稿に移動します。
レビュータブに移動します。
コメントフィールドに@の後にユーザー名を入力します。
適切なユーザーのリストが表示されます。
メンションしたいユーザーを選択します。
オプションで、コメントフィールドにさらに指示を書きます。
[ユーザー名]に割り当てるオプションを選択します。
タスクを割り当てる際には、1人のユーザーのみをメンションできます。
割り当てるをクリックします。
コメントがコメントフィールドの下に表示されます。
ユーザーにタスクを割り当てました。割り当てられたユーザーは、割り当てられたタスクに関するメールを受け取ります。メールにはニュース投稿へのリンクが含まれています。
割り当てられたタスクを完了した後、レビュアーはそれを完了としてマークできません。
コメントでの承認のリクエスト
コンテンツの承認は、組織の基準との一貫性を確保し、公開される投稿の質を向上させます。レビュータブ内で、責任のあるユーザーにワンクリックで承認をリクエストできます。
未完了の承認は、投稿の公開を妨げません。
スタジオで、承認をリクエストしたいニュース投稿に移動します。
レビュータブに移動します。
コメントフィールドに@の後にユーザー名を入力します。
適切なユーザーのリストが表示されます。
メンションしたいユーザーを選択します。
オプションで、コメントフィールドにさらに指示を書きます。
[ユーザー名]から承認をリクエストオプションを選択します。
承認をリクエストする際には、1人のユーザーのみをメンションできます。
リクエストをクリックします。
コメントがコメントフィールドの下に表示されます。
ユーザーに承認をリクエストしました。割り当てられたユーザーは、承認リクエストに関するメールを受け取ります。メールにはニュース投稿へのリンクが含まれています。
コメントでの承認リクエストの承認または拒否
メールボックスで、ニュース投稿に関するメールを見つけてクリックし、開くをクリックします。
ニュース投稿が新しいタブで開きます。
レビュータブに移動します。
リクエストされた承認のコメントに移動します。
クリック 承認 または拒否。
リクエストを承認しますか？またはリクエストを拒否しますか？ダイアログが表示されます。
クリック 承認 または拒否をクリックします。
あなたはニュース投稿のレビュータブで承認または拒否を行いました。承認をリクエストしたユーザーには、その結果についてのメールが送信されます。
