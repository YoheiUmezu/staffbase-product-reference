# 統合の紹介

**URL**: https://support.staffbase.com/hc/ja/articles/360015301659
**取得日**: 2026-05-02 18:47:42 JST

## 内容

統合とは何ですか？
統合は、Staffbaseプラットフォーム内の機能を豊かにしたり拡張したりします。これは、従業員アプリやイントラネット、またはスタジオ内の機能を拡張することによって行われます。
例えば：
外部コンテンツのリンクまたは埋め込み
ログイン保護された外部コンテンツの埋め込み
外部ソースとのコンテンツの同期
従業員アプリやイントラネット内で外部アプリケーションやプロセスを利用可能にする
ユーザーライフサイクル管理の自動化
これらのすべての機能は、Staffbaseプラットフォーム内でシームレスな体験を生み出します。
なぜ必要なのですか？
現代の組織は、日常業務のためにさまざまなツール、システム、アプリケーションを使用しています。さまざまなツールに分散されたコンテンツやサービスを中央の場所に集め、これらのツールを多くのユーザーに対して効率的に管理することは、しばしば困難です。統合は、異なるツールやシステム間のギャップを埋め、複数のプラットフォームにわたるユーザー管理の課題を解決します。Staffbaseを通じて、一般的なビジネスシナリオを統合できます。例えば：
ユーザー
異なるアプリケーションやツール間でユーザーを管理することは、現代の組織が巧みに対処しなければならないことです。異なるアプリケーションでのユーザー管理のためのさまざまなプロセスの統合は、これらのプロセスが自動的で信頼性が高く、効率的であることを保証します。
コンテンツ
他のアプリケーションやシステムからのコンテンツを従業員プラットフォームで利用可能にするか、その逆も可能です。ここでは、提供したいユーザー体験を定義することもできます。
例えば：
外部ソースからのコンテンツを橋渡ししたい場合は、リンクを選択します。
ユーザーにシームレスな体験を提供し、コンテンツを従業員アプリやイントラネットの一部として表示したい場合は、埋め込みを選択します。
プロセス
他のツールを活用するか、従業員アプリやイントラネットを通じてユーザーがプロセスを完了できるようにする既存の機能を使用することで、プロセスをStaffbaseプラットフォームに持ち込むことができます。
例えば：
ユーザーを引き付けたり、彼らの反応を得るために、投票やクイズを使用します。
SAPなどのHRシステムからの休暇承認プロセスを、従業員のモバイルアプリで利用できるようにします。
統合が設定されると、自動的でシームレスなワークフローが保証されます。
