# 従業員メールによるオープン数とクリック数のカウント

**URL**: https://support.staffbase.com/hc/ja/articles/4415079626514
**取得日**: 2026-05-02 18:53:58 JST

## 内容

従業員メールで追跡しているメールが受信者によって開かれたり、追跡されたメールの本文内のリンクがクリックされたりすると、このアクティビティが記録され、複数のウェブアプリでのメトリクスの表示を使用してアクセスできます。
この記事では、ユニークオープン + クリックとリピートオープン + クリックの違いを詳しく説明します。また、予期しないユニークオープン数が表示される理由や、ユニークオープンの膨張を説明または防ぐ方法についても説明します。
オープン
オープンは、受信者があなたのメールを読み込み、メッセージの本文を表示したときにトリガーされます。
特定の受信者がメールを初めて開いたとき、それはユニークオープンをトリガーします。
メールがアドインまたは拡張機能を使用して送信された場合、クッキーが受信者のデバイスに保存され、同じデバイスでそのメールが将来開かれるとリピートオープンとしてマークされます。
メールがウェブアプリまたはスタジオから直接送信された場合、受信者のユニークコンタクトIDがあなたの配信リストに保存されます。受信者が複数のデバイスで同じメールを開いても、デフォルトではこの人に対して匿名でカウントされるのは1つのユニークオープンのみで、残りはリピートオープンとしてカウントされます。
このようにして、あなたはメールを開いた受信者の数だけでなく、そのコンテンツを再訪した回数についても洞察を得ることができます。
クリック
追跡されたメールの本文内のリンクのクリックも、送信者がアドインまたは拡張機能の設定でリンクトラッキングを無効にしない限り、従業員メールによって自動的に追跡されます。
リンクが初めてクリックされたとき、オープンと同様に、それはユニーククリックを生成します。受信者がそれらのリンクを再度クリックすると、これはリピートクリックを生成します。
アドインまたは拡張機能を使用して送信されたメールでは、各デバイスでのユニーククリックとリピートクリックを追跡できます。
ウェブアプリまたはスタジオから直接送信されたメールでは、受信者がリンクを開くたびに1つのユニーククリックのみが追跡され、複数のデバイスを使用しても同様です。
一般的な懸念
ここで説明できない問題が発生した場合や調査が必要な場合は、Staffbaseサポートにお問い合わせください。私たちはいつでもお手伝いします！
低いユニークオープン
ユニークオープンの数が一貫して低い場合、受信者がメールを表示する際に問題がある可能性があります。
受信者がメールクライアントでコンテンツがブロックされているという視覚的警告を見ているか、権限が必要かどうかを調査する必要があります。
従業員メールでデザインされたメールに送信されたすべての画像は、私たちのサーバーにホストされます - これにより、あなたのメールが今だけでなく、将来も正常に機能し続けることが保証されます。あなたのネットワークが私たちのサーバーを許可リストに追加して、これらの画像が毎回読み込まれるようにすることが重要です。この問題は、あなたのコミュニケーションの一部でのみ発生する場合があります - その場合、特にオーディエンスに関して、これらの送信の何が異なるのかを考慮してください。
ゼロユニークオープン
送信したメールがユニークオープンを生成していない場合、通常はそうではない場合、作成者が以前に追跡された送信からコンテンツをコピーした可能性が高いです - 彼らは誤って古いトラッキングピクセルをメールに貼り付けたかもしれません。
これが一貫して発生する場合、あなたのネットワークで従業員メールのサーバーへの接続がブロックされている可能性が高いです。いずれにせよ、調査が必要です。
異常に高いオープン数および/またはクリック数
アドインまたは拡張機能による影響
時には、あなたのユニークオープンのカウントやユニーククリックのカウントが、送信先の受信者数よりも高くなることがあります。特にアドインや拡張機能を使用して送信する場合、結果はデバイスごとにカウントされ、各受信者が複数のデバイスを使用している可能性があるためです。組織内の人々が特にメールを読むのが得意な場合、これはほぼ常に当てはまります。
Staffbaseは、単一の送信を指標として使用するのではなく、時間の経過に伴うメトリクスのトレンドを分析することを推奨します。
ウェブアプリまたはスタジオから直接送信されたメールは、受信者がメールを開くたびに1つのユニークオープンと、各リンクに対して受信者ごとに最大1つのユニーククリックを追跡します。たとえ複数のデバイスを使用しても同様です。
セキュリティシステムによる影響
顧客の環境内の1つ以上のセキュリティシステムが、メールの内容をスキャンして有害なリンクや添付ファイルが含まれているかどうかをテストする場合があります。多くの場合、これらのサービスは信頼できるリンクの周りに追加のラッパーURLを追加します。この種のシステムが従業員メールを信頼するように設定されていない場合、メールメトリクスが異常に高いオープン数やクリック数を記録する原因となることがあります。
メールが送信された直後に、メール統計ページでの各メールのオープン率/クリック率が100％以上に急上昇し、または平均ユニークオープン率/クリック率が概要ページに表示される場合、主要なスパイクが見られます：
組織内の受信者の受信トレイで送信されたメールを開きます。
リンクにカーソルを合わせ、セキュリティシステムによって追加されたブランドリンクラッパーがあるかどうかを確認します。
たとえば、safelinks.protection.outlook.comやurldefense.proofpoint.comなど。
MS 365 Defender、Proofpoint URL DefenseおよびAttachment Defense、またはその他の同等のサービスで、組織のSafe LinksおよびSafe Attachmentsの設定を変更します。
送信前に、トラッキングドメインラッパーを含むリンクの例外を作成します。従業員メールがすべてのリンクに適用されます。
デフォルトおよびカスタムトラッキングドメインについて詳しく学び、従業員メールリンクをラップするために使用できます。
例外は、リンクが自動的にスキャンされ、受信者に届く前に追加のメールオープンやクリックを登録しないようにするためのものです。また、リンクに新しいラッパーが追加されるのを防ぐためでもあります。
非常に低いリピートオープン
これは低いエンゲージメントが原因かもしれませんが、5%未満の割合を見ると、問題がある可能性が高いです。35%のリピート率は一般的に健康的でおおよそ平均と見なされますが、これは組織によって大きく異なる場合があります。
グループポリシーがクッキーをブロックするように設定されているかどうか、ITチームに確認してください。
リンククリックなし
追跡されたメールにリンクが含まれているがクリックが記録されていない場合、最初に確認すべきことは、送信者がアドインまたは拡張機能の設定でリンクトラッキングを無効にしているかどうかです。
有効になっている場合、問題の原因はリンク自体にあるかもしれません。受信したメールのリンクを確認し、トラッキングプロキシが追加されていることを確認してください。
例: http://s.bl-1.com/h/c5bzc7K2?url=http://www.google.com
米国ホスティングの一部の顧客は、上記のデフォルトの米国ドメインの代わりに、カスタムトラッキングドメインを設定することを選択します。
EUホスティングの顧客の場合、トラッキングドメインには.de.bl-1.comが含まれます。
トラッキングプロキシがない場合、メトリクスは生成されません。
また、url=の後のリンクをブラウザにコピー＆ペーストしてみてください。読み込みに問題がある場合、メールで使用したリンクが壊れている可能性があります。送信前にすべてのリンクをテストして、送信後に機能することを確認するのが良い習慣です。
