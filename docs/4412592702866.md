# Microsoft Entra IDのカスタムフィールドをメール用に生成する

**URL**: https://support.staffbase.com/hc/ja/articles/4412592702866
**取得日**: 2026-05-02 18:51:57 JST

## 内容

製品名に関する注意: 2022年以降、BananatagはStaffbaseのEmployee Email製品です。
一部の重要な側面は、app.bananatag.comなどのように、一時的にBananatagの名前を保持しています。
実質的に、BananatagとEmployee Emailは同義です。
要件
組織のカスタムフィールドをBananatagに追加する前に、いくつかの要件があります:
組織のMicrosoft Entra IDはオンプレミスであり、Microsoft Entra (以前のAzure AD)に同期されています。
あなたのMicrosoft Entra IDのグローバル管理者がカスタムフィールドの生成を支援できる必要があります。
カスタムフィールドの特定
組織のMicrosoft Entra IDを開始するには、グローバル管理者が以下のスクリプトをPowershellで実行して、組織が内部で使用しているユニークなカスタムフィールドを取得する必要があります:
Get-AzureADUser -SearchString user@domain.com | select -ExpandProperty extensionproperty
上記の情報は、このクエリで返されるべきです...
「Prefix」以外の各セクションの拡張値は、組織に特有であり、以下を含みます:
AppClientID - 組織のユニークなMicrosoft Entra IDテナントID。
DisplayName - 組織のオンプレミスアクティブディレクトリで設定されたカスタムフィールドの表示名。
Value - 検索されたユーザーのこのカスタムフィールドの値。
上記の例では、Bananatagのオンプレミスアクティブディレクトリに「isBtagOnPrem」という名前のフィールドがあり、真または偽の値を受け入れます。
カスタムフィールドのインポート
カスタムフィールドがわかったら、それらをBananatagにインポートできます。これにより、これらのフィールドの基準に従ってリストをセグメント化したり、メールメトリクスをセグメント化したりできます。
1. カスタムフィールドのインポートをクリックします。
2. ダイアログが開きます。カスタムフィールドを挿入します。フィールドを追加をクリックすることで、複数のフィールドを追加できます。
3. すべてのカスタムフィールドを追加したら、カスタムフィールドのインポートをクリックして保存します。
4. マッピングページでカスタムフィールドが表示されます。次に、それらをBananatagフィールドにマッピングできます。
