# Microsoft 365 接続のためのカスタムアプリの設定

**URL**: https://support.staffbase.com/hc/ja/articles/4410110692370
**取得日**: 2026-05-02 18:58:21 JST

## 内容

管理者
Azure アプリを自分で構成および管理したい場合は、Azure ポータルでカスタムアプリを作成する必要があります。以下の手順に従って、カスタムアプリを作成してください：
この記事では：
Azure でのカスタムアプリの作成
Microsoft 接続のためのクライアント資格情報の取得
カスタム Microsoft 365 接続へのリダイレクト URI の追加
前提条件
あなたは Azure グローバル管理者です
Azure でのカスタムアプリの作成
Azure ポータルで、Microsoft Entra ID > アプリ登録に移動します。
+ 新しい登録をクリックします。
アプリの名前を入力します。
アカウントの種類として任意の組織ディレクトリのアカウント (任意の Azure AD ディレクトリ - マルチテナント)を選択します。
登録をクリックします。
アプリが作成され、詳細が表示されます。
Microsoft 接続のためのクライアント資格情報の取得
カスタム Microsoft アプリのクライアント ID とクライアント シークレットの両方が必要です。これにより、Staffbase プラットフォームでカスタム Microsoft 接続を正常に構成できます。
Microsoft 接続の構成を別のタブで開いておくと、クライアント資格情報を簡単にコピーして貼り付けることができます。
カスタム Microsoft アプリの Azure アプリ クライアント ID の取得
Azure ポータルで、Microsoft Entra ID > アプリ登録に移動します。
Microsoft 接続のために作成したアプリを選択します。
アプリケーション (クライアント) ID をクリップボードにコピーします。
Staffbase Studio の Microsoft 接続ダイアログのAzure アプリ クライアント IDフィールドにクライアント ID を貼り付けます。
Azure アプリ クライアント シークレットの作成
Azure ポータルで、Microsoft Entra ID > アプリ登録に移動します。
Microsoft 接続のために作成したアプリを選択します。
クライアント資格情報のために証明書またはシークレットの追加をクリックします。
証明書とシークレットページが開きます。
+ 新しいクライアント シークレットをクリックします。
説明と有効期限を入力します。
クライアント シークレットの有効期限が切れた後、Microsoft 接続は Azure で更新する必要があります。
追加をクリックします。
値をクリップボードにコピーします。
Staffbase Studio の Microsoft 接続ダイアログのAzure アプリ クライアント シークレットフィールドにシークレット ID を貼り付けます。
Staffbase リダイレクト URI を Microsoft Azure アプリに追加
従業員アプリまたはイントラネットは、作成したカスタム Azure アプリに接続する必要があります。Staffbase リダイレクト URI をカスタム Azure アプリに追加する必要があります。
前提条件
カスタム Azure アプリを作成しています。
カスタム Azure アプリを使用して Studio で Microsoft 接続を構成しています。
Staffbase プラットフォームがホストされている場所を知っています。詳細についてはこの記事をお読みください。
Microsoft Entra ID (旧 Microsoft Azure Active Directory) で、アプリ登録に移動します。
登録したアプリで、リダイレクト URI の追加をクリックします。
プラットフォームの追加をクリックします。
Webを選択します。
Web アプリケーション URI を構成するためのダイアログが開きます。
設定に基づいて、適切なリダイレクト URI をリダイレクト URIフィールドに貼り付けます。
デフォルトの Azure アプリ：
ホスティングアーキテクチャに基づいて：Staffbase プラットフォームが使用している場合
米国ホスティングインフラストラクチャ：
https://backend-us1.staffbase.com/api/integrations/auth/ms365/callback
ドイツホスティングインフラストラクチャ：
https://backend-de1.staffbase.com/api/integrations/auth/ms365/callback
カスタム Azure アプリ：
https://{your_app_domain}/api/integrations/auth/ms365/callback
構成をクリックします。
保存をクリックします。
これにより、構成されたリダイレクト URI が確認されます。
カスタム Azure アプリにリダイレクト URI を追加しました。これで、ユーザーは Staffbase プラットフォームから Microsoft 365 にサインインできるようになります。
