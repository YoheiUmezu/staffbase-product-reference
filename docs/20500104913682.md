# Box統合の設定

**URL**: https://support.staffbase.com/hc/ja/articles/20500104913682
**取得日**: 2026-05-02 18:46:31 JST

## 内容

ベータ
アクティベーション
StaffbaseとのBox統合により、ユーザーはプラットフォーム全体でコンテンツを見つけるのがこれまで以上に簡単になります。このFront Door Intranet用の統合を使用すると、Staffbaseプラットフォーム内のコンテンツだけでなく、Boxのコンテンツにもアクセスできます。
前提条件
あなたはBox管理者です。
StaffbaseがBoxインスタンスにアクセスするためのエンドポイントの生成
統合を開始する前に、StaffbaseがBoxインスタンスにアクセスするためのエンドポイントを生成する必要があります。
Box開発者コンソールで、マイアプリに移動します。
新しいアプリを作成をクリックします。
新しいアプリを作成するページが開きます。
カスタムアプリを選択します。
カスタムアプリを作成するためのダイアログが開きます。
必須フィールドに入力します。
Staffbaseは以下の情報の使用を推奨します：
目的：統合
カテゴリ：生産性
統合する外部システムは？：Staffbase
次へをクリックします。
ユーザー認証 (OAuth 2.0)オプションを選択します。
アプリを作成をクリックします。
StaffbaseがBoxインスタンスにアクセスするためのエンドポイントを生成しました。これで、Boxの設定を構成し、クライアントIDとクライアントシークレットをコピーして、StaffbaseにBox統合を追加できます。
Boxインスタンスの構成
Box接続をプラットフォームに追加する前に、Boxインスタンスを構成する必要があります。
Box開発者コンソールで、マイアプリ > 作成したアプリに移動します。
構成タブを選択します。
OAuth 2.0リダイレクトURIまでスクロールします。
リダイレクトURLフィールドに、次の内容を入力します： https://[YOUR_STAFFBASE_DOMAIN]/api/integrations/auth/box/callback
たとえば、ドメインがmy_staffbaseの場合、https://my_staffbase/api/integrations/auth/box/callbackと入力します。
追加をクリックします。
OAuth 2.0資格情報までスクロールします。
クライアントIDとクライアントシークレットをコピーします。
変更を保存をクリックします。
Boxの設定を構成し、クライアントIDとクライアントシークレットをコピーしました。これで、Box接続をStaffbaseに追加できます。
