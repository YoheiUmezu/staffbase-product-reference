# Staffbase内でのHR統合の設定

**URL**: https://support.staffbase.com/hc/ja/articles/27950129576978
**取得日**: 2026-05-02 18:51:00 JST

## 内容

有効化
管理者
Staffbaseは、HRシステムとプラットフォーム間のシームレスな同期を可能にします。組織の構造やビジネスニーズに合わせて、1つまたは複数のHR統合を設定できます。
前提条件
統合したいHRシステムで管理者またはパワーユーザーの役割を持っています。
または、統合設定時にHRシステムの管理者やパワーユーザーと一緒に作業できます。
外部HRサービスの準備ができています。手順については、ご利用のHRシステムのセットアップガイドをご参照ください。
SAP Success Factors
Workday
ウィジェットの利用を決定しています。
欠勤
給与明細
統合用のユーザー識別子を用意しています。例：仕事用メール、パブリックメール、リモートIDなど。
StaffbaseでHR統合を追加する
Studioで設定 > HR 統合に移動します。
+ 統合を追加をクリックします。
希望する統合を選択します。
追加するをクリックします。
ユーザー識別子を選択ダイアログが開きます。
Staffbaseスタジオと選択したサービスのドロップダウンから、ユーザーデータをマッピングする識別子を選択します。例：リモートIDにはSAP SuccessFactorsのpersonalIdExternalフィールドが入力されます。
次をクリックします。
以下の情報を入力し、それぞれ入力後に送信をクリックします。
API サーバーURL
ユーザーネーム
Company ID
クライアント ID
クライアントシークレット
HR統合がStudioに追加されました。編集者は有効化されたHRウィジェットをページに追加し、従業員が自分のページで情報を閲覧できるようにできます。統合が完了すると、HRサービスの関連機能はデフォルトで有効化されます。いつでも無効にすることができます。
データ同期の確認
統合が設定されると、HR統合名の横に更新のステータスが表示されます。ステータスは24時間ごとに自動で同期されます。以下のように表示されます：
同期済み：統合は有効で、データは最新です。
未完了：接続が正しく確立されていません。
ステータスをクリックすると、ユーザーがアクセスできる個別のデータフィールドを確認できます。以下のように表示されます：
完了：フィールドが同期されています。
失敗：フィールドが完全に同期できませんでした。
HR統合が追加され、データが同期されると、HR統合名の横に「同期済み」ステータスが表示されます。同期ステータスを確認し、必要に応じて再同期を実行できます。
