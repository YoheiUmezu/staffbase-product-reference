# 従業員メールOutlookアドインからのエイリアスのトラブルシューティング

**URL**: https://support.staffbase.com/hc/ja/articles/4415079647890
**取得日**: 2026-05-02 18:59:43 JST

## 内容

内部コミュニケーションでは、コミュニケーターがリーダーの名義で、または共有メールボックスから送信することが標準となっており、これによりコミュニケーション戦略内での一貫性、信頼、ブランディングを助けています。
Employee Emailアドインを使用してOutlook 365またはOutlook Classicから複数のメールアドレスで送信する際に、エラーが発生することがあります。
このガイドでは、これらの問題を自分で解決するための手順を説明するか、Outlook 365またはOutlook Classicのいずれを使用しているかに応じて正しい方向に案内します。
Outlook 365
続行する前に、すべてのエイリアスメールボックスにEmployee Email内でアカウントが作成されており、正しいサブスクリプションにリンクされていることを確認してください。
ネイティブOutlook作成ウィンドウを使用して作成されたメールの場合、作成ウィンドウ内のFrom フィールドのドロップダウンメニューを変更するだけで、Track & Send ボタンを選択してメールを追跡できます。
メールデザイナーを使用して作成されたメールの場合、O365アドイン送信ウィンドウのFrom ドロップダウンメニュー内のAdd New Alias オプションを使用して追加のエイリアスを追加することができます。

Outlook Classic
Outlook Classic用のアドインは、すでに送信権限を設定したすべてのメールボックスを採用しますが、アドイン内でドラフトを送信しようとしているときにFrom フィールドにメールボックスがオプションとして表示されない場合、次のステップはアドインを閉じ、新しいメールをOutlookで開き、そのメールボックスから自分に未追跡のメールを送信することです。
これが機能する場合、再度アドインに入ると、そのメールボックスが利用可能になるはずです。そうでない場合は、Outlookのホーム画面からReport Bugをクリックしてサポートに連絡してください。
他のメールボックスからメールを送信しようとしたときに一般的なエラーが発生した場合は、まず個人アカウントを選択し、自分に未追跡のメールを送信してテストしてください。エラーがまだ発生する場合は、別の原因が問題を引き起こしている可能性があるため、Outlookのホーム画面からReport Bugをクリックしてサポートに連絡してください。
