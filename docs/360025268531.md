# ページおよびその他のプラグインの役割

**URL**: https://support.staffbase.com/hc/ja/articles/360025268531
**取得日**: 2026-05-02 18:50:27 JST

## 内容

ページ、イベント登録、地図などのプラグインを追加するには、システム全体の役割が必要です。その後、プラグインにコンテンツを追加し、プラグインを管理するために、プラグイン固有の役割を割り当てることができます。

役割の詳細
ユーザー
グループ管理とコンテンツの可視性設定を使用することで、特定のユーザーのニーズに合わせたコンテンツをターゲットにすることが可能です。たとえば、特定の場所や興味グループに向けてです。ターゲットにされていないユーザーは、ページやプラグインを表示しません。
ターゲットにされたユーザーは、自分または自分のグループのために公開されたコンテンツを表示する権利を持っています。ユーザーはページやプラグインを見るだけでなく、ほとんどのプラグインはユーザーに対して、イベントへの登録や満足度の評価などのインタラクションを求めます。
ページエディター / プラグインエディター
エディターは、ページやプラグインを管理する役割です。これには、編集、公開、非公開が含まれます。この役割は、ページやプラグインの管理権限を付与するのに最適ですが、削除したり他のものを追加したりすることはできません。
ページ所有者
ページ所有者は、ページの編集、公開、非公開、ページタイトルや説明の調整ができる役割です。この役割は、ページ コンテンツの正確性とコンプライアンスの維持を担当します。更新や質問のための専任の連絡先を明確にすることで、コンテンツ ガバナンスを確実にします。
グループ管理者
グループ管理者はグループメンバーの追加と削除、グループ設定の調整の権限を持っています。グループ管理者はコンテンツではなく、ユーザーに焦点を当てています。
スペース管理者
スペース管理者はスペース全体の管理権限を持ち、ページやプラグインの設定を行います。
システム全体のエディター
システム全体のエディターは通常のエディターよりもはるかに多くの権限を持ち、プラットフォーム内のシステム全体のページやプラグインにアクセスし、利用することができます。管理エディターとの唯一の違いは、システム全体のエディターは新しいページやプラグインを追加できないことです。
管理エディター
管理エディターは、プラットフォーム内のシステム全体のページやプラグインを管理するために必要な完全な権限を持っています。システム全体のエディターに加えて、この役割は新しいページやプラグインを追加することができます。
管理者
管理者はシステム全体の管理権限を持ち、ページやプラグインの設定を含みます。
Staffbaseは、管理者がユーザー管理やプラットフォームの設定にもアクセスできるため、コンテンツ管理にはこの役割を使用しないことを推奨しています。
