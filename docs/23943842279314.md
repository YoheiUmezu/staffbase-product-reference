# リリースサイクルの概要

**URL**: https://support.staffbase.com/hc/ja/articles/23943842279314
**取得日**: 2026-05-02 18:42:56 JST

## 内容

Staffbaseは、従業員とのコミュニケーションを向上させるために、新機能、改善、バグ修正を常にリリースしています。継続的なコードデプロイメント、機能管理、四半期ごとの機能リリースの組み合わせを活用することで、Staffbaseはプラットフォームに合わせたシームレスな更新の流れを確保しています。
この記事では、継続的なコードデプロイメントと四半期ごとのリリースを含むリリースプロセスと、それがプラットフォームに与える影響について説明します。
継続的コードデプロイメント
継続的コードデプロイメントにより、Staffbaseはバグ修正、セキュリティ強化、マイナーな改善を含む更新を頻繁に提供できます。Staffbaseプラットフォームには、異なるリリースサイクルに従うさまざまなコンポーネントがあり、1日に複数回から週ごとのスケジュールまでさまざまです。重要なバグやセキュリティ修正のためにアドホックリリースが行われます。このアプローチにより、複数のチームが同時に機能に取り組むことができます。
Staffbaseは、これらの変更を隔週で最新のStaffbaseの更新を通じて通知します。
Staffbase APIについては、新しいAPIのリリースや既存のAPIの更新が開発者の変更履歴を通じて通知されます。
四半期ごとの機能リリース
Staffbaseは、年に4回の主要な機能リリースを実施しています。これらのリリースは、それぞれの四半期と年に基づいて名付けられ、例えば「Q1 2025リリース」のようになります。各機能リリースでは、ウェブおよびモバイルアプリケーション向けの新機能と改善が導入されます。
四半期ごとの機能リリースのすべての新機能と改善も、継続的なコードデプロイメントに従います。これは、日次および週次のリリース中に段階的にデプロイされますが、プラットフォーム上では非表示のままです。このため、Staffbaseは機能フラグを使用して、プラットフォーム上で何が表示されるかを制御し、ユーザーエクスペリエンスを損なうことなくシームレスな継続的開発を確保します。このアプローチにより、Staffbaseは変更の事前通知を行うことも可能です。
公式リリース日には、Staffbaseはプラットフォーム上で利用可能にする必要がある機能を有効化します。製品ライフサイクルの段階に応じて、一部の機能はStaffbaseサポートまたはカスタマーサクセスマネージャーにリクエストすることでのみ利用可能です。
機能がすべての顧客に自動的に展開される場合、Staffbaseは機能フラグを削除して、すべての顧客に対してその機能を有効化します。
Staffbaseは、これらのリリース機能について、リリースノートを通じて2週間前に通知し、ユーザーへの変更を準備または伝える時間を提供します。さらに、Staffbaseはリリースノートが準備できたことを顧客に知らせるためのメールを送信します。このメールを受け取りたい場合は、カスタマーサクセスマネージャーまたはStaffbaseサポートにお問い合わせください。
継続的デプロイメントにより、Staffbaseはプラットフォームを一貫して最新の状態に保ち、四半期ごとのリリースは内部コミュニケーションを簡素化し、強化するための革新的な機能を提供します。
