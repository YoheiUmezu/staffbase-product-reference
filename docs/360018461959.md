# 分析におけるデータ保護

**URL**: https://support.staffbase.com/hc/ja/articles/360018461959
**取得日**: 2026-05-02 18:47:38 JST

## 内容

測定できないものは管理できません。これは、コミュニティやオーディエンスに関して特に当てはまります。Staffbaseは、あなたとあなたのユーザーの正当な利益のために、ニュースやその他のコンテンツのリーチと効率を測定し、向上させることが重要であると考えています。これにより、ユーザーにとって魅力的で関連性のあるコンテンツを提供できるようになります。Staffbaseにとって、エンゲージメントを測定するツールは、プライバシーとデータ保護に強く焦点を当てることが不可欠です。
Staffbaseは、スタジオでの分析提供のために限られた個人データを処理します。Staffbaseは、データ処理者として一般データ保護規則（GDPR）の基準を適用し、適用されるデータ処理契約（DPA）に従ってのみ個人データを処理します。あなたは、組織の分析提供のために処理される情報をコントロールし続けます。
従業員のプライバシーを重視しています
Staffbaseは、製品、サービス、内部ガバナンスの開発の中心にプライバシーとデータ保護を置いています。これは、分析の構築方法にも当てはまります。このため、以下のことを確実に行っています：
第三者の分析サービスは使用しません：Google Analyticsなどの第三者のサービスは使用していません。私たちは、分析を社内で構築し管理しており、コンテンツの価値ある洞察を提供します。
データ最小化：分析提供のために、Staffbaseはユーザー活動に関連する限られた情報のみを処理します。分析メトリクスを作成するために必要な情報には以下が含まれます：
時間に基づいて活動をフィルタリングするためのタイムスタンプ。
ユニークなユーザーメトリクスを決定するためのユーザーID。
ユーザーIDは、他のユーザー属性（ユーザー名など）とは無関係なランダムな数字のセットです。各ユーザーアカウントの作成時にStaffbaseによって生成され、私たちのシステムに安全に保存されます。ユーザーIDを分析の利益のために使用することで、ユーザーエンゲージメントの正確な状況を把握できます。
ユーザープロファイルの追跡は行いません：分析は特定のユーザーのステップを追跡するための個別のユーザープロファイルを作成しません。代わりに、これらの個別の活動は一般的なユーザーインタラクションに関する有用なレポートに集約されます。たとえば、スタジオでは、月曜日の午後に投稿を作成したユニークユーザーの数を見ることができますが、どのユーザーであったかはわかりません。
オプトインおよびオプトアウトのオプション：リクエストに応じて、Staffbaseは分析のためのオプトインまたはオプトアウトオプションを有効にできます。オプトインオプションでは、ユーザーはログイン時に一度、使用データの機密分析に同意するよう求められ、アプリの体験を向上させます。オプトアウトオプションでは、ユーザーは自分のアプリの設定から分析の目的でのユーザーIDの使用に異議を唱えることができます。オプトインおよびオプトアウトオプションに関する詳細については、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
これらのオプションのいずれかが有効になった場合、あなたの分析メトリクスはもはや全ユーザーベースを反映しなくなることに注意してください。
ユーザーへの透明性：あなたは、アプリやイントラネットのプライバシーポリシーを組織のニーズや法的要件に合わせてカスタマイズすることもできます。これにより、スタジオで分析を使用する際の目的と範囲について明確で役立つ情報を提供できます。
分析に関する質問がある場合は、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
