# 最新の従業員メール Outlook 365 アドインの承認を取得する

**URL**: https://support.staffbase.com/hc/ja/articles/5973812214546
**取得日**: 2026-05-02 18:49:37 JST

## 内容

製品名に関する注意: 2022年以降、BananatagはStaffbaseの従業員メール製品です。
最新の従業員メール Outlook 365 アドインにアクセスするには、あなたのMicrosoft Entra ID（以前のAzure AD）管理者が、以前のアドインバージョンでは必要なかった以下のスコープを承認する必要があります:
mail.readwrite.shared
mail.send.shared
最新のアドインバージョンに切り替える際に、承認が必要というダイアログが表示された場合、承認を得る方法は2つあります。
Microsoft Entra ID 管理者に承認をリクエストする
Microsoft Entra ID 管理者の資格情報でサインインする
Microsoft Entra ID 管理者に承認をリクエストする
権限のリストが表示された承認が必要ダイアログで、承認をリクエストをクリックします。
あなたのMicrosoft Entra ID管理者がリクエストを受け取ったことを確認します。
彼らは、Microsoft Entra管理ポータルにアクセスし、エンタープライズアプリケーション > 管理者の同意リクエストに移動することで、リクエストを表示し、承認できます。
ポータルにリクエストが表示されない場合は、Microsoft Entra ID管理者に、承認が必要ダイアログで直接管理者の資格情報でサインインするように依頼してください。
Microsoft Entra ID 管理者の資格情報でサインインする
この方法では、Microsoft Entra ID 管理者がサインインし、最新の Outlook 365 アドインのためにあなたの組織を代表して権限を承認する必要があります。
権限のリストが表示された承認が必要ダイアログで、あなたのMicrosoft Entra ID管理者に別のアカウントでサインインをクリックするように依頼します。
Microsoft Entra ID 管理者の資格情報を入力してサインインします。
リクエストを受け入れるためのダイアログが開きます。
あなたの組織を代表して同意するにチェックを入れます。
受け入れるをクリックします。
あなたの Microsoft Entra ID 管理者があなたの組織を代表してリクエストを承認しました。
これにより、すべてのユーザーが独立してアップグレードし、Bananatag Service APIによって要求された権限を受け入れることができます。
