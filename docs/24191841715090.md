# Staffbase Emailにおけるユーザーグループの概要

**URL**: https://support.staffbase.com/hc/ja/articles/24191841715090
**取得日**: 2026-05-02 18:55:23 JST

## 内容

Staffbase EmailをStaffbase IntranetまたはEmployee Appと一緒に使用する場合は、グループの種類を参照してください。以下の情報は、Staffbase Emailがスタンドアロン製品として使用される場合にのみ適用されます。
ユーザーグループを使用すると、場所、部門などのプロファイルフィールド情報に基づいてユーザーをクラスタリングできます。
ユーザーグループは、以下のユースケースにとって不可欠です：
ターゲットを絞ったメールコミュニケーション：特定のユーザーグループが特定のメールを受信するように定義することで、パーソナライズされた関連性のあるメッセージを送信します。
アクセスと権限の管理：個々のユーザーではなく、グループに権限を割り当てることで、スタジオでのユーザー管理を簡素化します。たとえば、複数のユーザーを手動で追加するのではなく、グループにフォルダーアクセスを付与できます。

グループメンバーシップを決定する条件を定義することで、ユーザーグループを作成できます。グループは、次のロジックを使用して1つまたは複数の条件で設定できます：
AND：指定されたすべての条件を満たすユーザーのみがグループの一部となります。
OR：指定された条件のいずれかを満たすすべてのユーザーがグループの一部となります。
これらのグループを使用して、メールターゲティングのニーズに応じた正確または広範なグループを作成できます。
例
シナリオ1：
特定の場所にあるマーケティング部門のユーザーグループを作成するには、ANDロジックを使用して他のすべてのユーザーを除外します。
シナリオ2：
マーケティング部門または営業部門のユーザー、またはCEOやVPなどの役職にあるユーザーのグループを作成するには、ORロジックを使用して、これらの条件のいずれかを満たすユーザーを含めます。
手動で
個々のユーザーを手動で選択することでユーザーグループを作成できます。この方法はより多くの労力を要しますが、ユーザーグループを迅速に作成または変更する必要がある場合に完全な柔軟性を提供します。このアプローチは、一度限りまたはアドホックなグループ設定に最適です。
Entra ID統合
Microsoft Entra IDに存在するユーザーグループを同期できます。Entra ID統合を設定した場合、これらはStaffbase Emailでの連絡先の同期にのみ使用できます。この統合により、ユーザーグループは自動同期を通じて常に最新の状態に保たれます。
Entra IDから同期されたグループは、Staffbaseプラットフォームで編集できません。これらのグループに対する変更はEntra IDで行う必要があります。
ユーザーグループを作成した後、ユーザーのプロファイルフィールド情報がグループの条件に一致する場合、ユーザーは自動的に適切なグループに割り当てられます。これにより、ユーザーが追加されたり、詳細が変更されたりする際にグループが最新の状態に保たれます。
