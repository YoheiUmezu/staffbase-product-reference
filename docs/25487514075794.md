# プラグインの概要

**URL**: https://support.staffbase.com/hc/ja/articles/25487514075794
**取得日**: 2026-05-02 18:42:09 JST

## 内容

プラグインは、Staffbaseプラットフォームに特定の機能を追加できるミニアプリケーションです。これらはStaffbaseプラットフォームにシームレスに統合できます。ほとんどの場合、プラグインはプラグアンドプレイの形式で開発されていますが、特定の外部システムとStaffbaseプラットフォーム間の通信を確立するために、コネクタを選択する必要がある場合もあります。
↻
プラグインの種類
Staffbaseは、プラットフォームの機能を拡張するためのさまざまな可能性を提供しています。プラグインは大きく分けて以下の2種類に分類できます：
標準プラグイン
これらはStaffbaseが提供するすぐに使用できるプラグインで、さらに以下のように分類できます：
プラットフォームに埋め込まれたStaffbase専用のWebアプリケーション
例：Formsプラグイン
サードパーティのサービスやシステムと統合されたStaffbaseのWebアプリケーション
例：Microsoft 365 Filesプラグイン
カスタムプラグイン
これらは特定のビジネスユースケースを簡素化するためにカスタム構築されています。これらは社内で開発することも、パートナーやベンダーに外注することもできます。カスタムプラグインの開発について詳しくは、こちらをご覧ください。
プラグインを介してコンテンツを利用可能にする
プラグインの種類に関係なく、管理者と管理編集者は、同じプラグインを使用してさまざまなコンテンツをユーザーに提供することができます。これを3つの簡単なステップで行います：
1. インストール
管理者と管理編集者は、組織に利用可能なすべてのプラグインをインストールできます。スタジオのプラグインページをブラウズして、組織に利用可能なプラグインを確認してください。追加のプラグインを組織に対して有効化するには、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
2. 追加
プラグインがインストールされると、管理編集者は同じプラグインの異なるインスタンスを追加して、関連するユーザーにさまざまなコンテンツやプロセスを提供できます。たとえば、Formsプラグインをインストールした場合、フィードバック用のフォームやインシデント報告用のフォームなど、Formsプラグインの異なるインスタンスを使用して異なるフォームを追加できます。
3. 公開
プラグインコンテンツが準備できたら、プラグインの可視性設定に従ってユーザーに利用可能にするために公開します。ユーザーは、プラグインを使用していることに気づかない場合があります。なぜなら、プラグインはプラットフォームの一部として機能するからです。これらのプラグインは、Staffbaseプラットフォームでの使用に最適化されています。サードパーティのサービスと統合されたプラグインやカスタムプラグインの場合、ユーザーはStaffbaseプラットフォームの外部にホストされているコンテンツにアクセスするために追加のサインインを行う必要はありません。これは、バックグラウンドで行われるシングルサインオンフローによって可能であり、データは認可されたユーザーのみがアクセスできることを保証します。
ユーザーがプラグインを使用して異なるコンテンツを表示する方法
この例では、Formsプラグインの異なるインスタンスがユーザーに提供されています。
