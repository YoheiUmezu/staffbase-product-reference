# ユーザーグループの種類

**URL**: https://support.staffbase.com/hc/ja/articles/360010803979
**取得日**: 2026-05-02 18:47:48 JST

## 内容

この記事の情報は、Employee App または Staffbase Intranet を使用している場合、かつ Staffbase Email がアプリまたは Intranet と一緒に使用されている場合に適用されます。Staffbase Email を単独製品として使用している場合は、Staffbase Email におけるユーザーグループの概要を参照してください。
ユーザーグループは、Staffbase プラットフォームの中心的な要素です。これにより、同じオフィスの場所、部門、または類似の興味を持つユーザーをクラスター化し、これらの類似性に特化したコンテンツをターゲットにすることができます。これは、さまざまな種類のユーザーグループを使用して、いくつかの方法で実行できます。
手動内部グループ
手動内部グループは、ユーザーには表示されません。ユーザーはグループに登録したり、登録を解除したりすることはできず、多くの場合、自分がグループにいることすら認識していません。各ユーザーは手動でグループに追加または削除されるため、メンバーの入れ替わりが少ない小規模なグループに最適なオプションです。
例:
Showcase, Inc. の部門長は、戦略的なトピックを互いに共有できるグループを望んでいます。このグループは小規模であり、より機密性の高い情報を含むため、手動管理は、ユーザーが複雑な CSV インポートを扱うことなくグループ内での自律性を持つための効果的なオプションです。
条件付きグループ
手動内部グループと同様に、条件付きグループもユーザーには表示されません。ユーザーはグループに登録したり、登録を解除したりすることはできず、CSV インポートやユーザー API などの自動形式を使用して管理する必要があります。条件付きグループでは、複数のタグに基づいてグループを設定できます。タグは、「And」および「Or」条件を設定して、ターゲットグループを広げたり狭めたりすることができ、コンテンツターゲティングをカスタマイズできます。
例:
Showcase, Inc. のマーケティング部門は、部門のメンバーが部門特有のコンテンツを受け取ることを保証するために条件付きグループを作成しました。「マーケティング」というタグがプロフィールに適用された各ユーザーは、自動的に条件付きグループに追加されます。このタグを持つユーザーのみがグループのコンテンツを見ることができます。
オープングループ
オープングループは、ユーザーが興味を積極的に追求し、類似の興味を持つ他のユーザーとつながる方法を提供します。ユーザーはオープングループに自由に登録および登録解除できます。これにより、興味グループなど、従業員のイニシアティブとエンゲージメントを促進するのに最適です。ユーザーがグループから登録解除することで、職務遂行に重要な情報へのアクセスを失う場合、そのコンテンツは別のタイプのグループに適している可能性があることを考慮することが重要です。
例:
オープングループ「Showcase Inc. ランナー」は、異なる部門やリーダーシップレベルの従業員が個人的に接続し、ランニングへの情熱を共有する機会を提供します。このグループでは、ユーザーが地域の今後のランニングイベント、グループイベントの写真を共有し、グループチャットでお互いにヒントを与えます。
必須グループ
必須グループは、特別なタイプのオープングループであり、必須オープングループとも呼ばれます。これらのグループには、自由に登録解除できない必須ユーザーのコアがあります。他のすべてのユーザーにとって、このグループは通常のオープングループのように機能します。これは、部門間の透明性を促進したい企業にとって便利です。必須グループを使用する際は、より大きなオーディエンスに表示されるため、機密情報を共有しないように注意することが重要です。このグループタイプは、CSV インポートまたはユーザー APIを介して管理する必要があります。
例:
必須オープングループ「マーケティング」は、すべてのマーケティング従業員にとって必須です。しかし、Showcase, Inc. は部門の活動を透明にし、マーケティンググループをすべての関心のある従業員に利用可能にしたいと考えています。たとえば、ファイナンス部門の従業員は、このグループに自由に登録および登録解除できます。
