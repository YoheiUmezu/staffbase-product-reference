# 多言語プラグインの作成

**URL**: https://support.staffbase.com/hc/ja/articles/360001597591
**取得日**: 2026-05-02 18:52:35 JST

## 内容

複数の言語でプラグインコンテンツを提供する
以下のプラグインは多言語コンテンツをサポートしています：
フォーム
イベント登録
カレンダー
クイズカレンダー
多言語コンテンツをサポートしていない他のプラグインについては、必要な言語でプラグインの一部コンテンツを提供する機会を与える回避策があります。
食事プランなどの一部のプラグインでは、現在複数の言語でコンテンツを提供するサポートがありません。ただし、これらのプラグインでは、インターフェースの大部分を自分でラベル付けすることができます。これにより、必要な言語でプラグインコンテンツを書く機会が得られます。
多言語サポートのあるプラグインのための多言語コンテンツの作成
別の言語でコンテンツを追加したいプラグインに移動します。
+ボタンをクリックして言語を追加します。
ドロップダウンメニューから言語を選択します。
ドロップダウンメニューには、組織で有効になっているすべての言語が含まれています。追加の言語が必要な場合は、管理者がそれらを有効にできます。
オプションで、既存のコンテンツをコピーのトグルを有効にして、別の言語から既存のページコンテンツをコピーします。
OKをクリックします。
新しい言語のタブが表示されます。
多言語サポートのないプラグインのための多言語コンテンツの作成
以下の方法で複数の言語のプラグインコンテンツを提供するには、言語ごとにユーザーグループを作成する必要があります。具体的な使用ケースについては、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
多言語サポートのないプラグインのコンテンツを複数の言語で作成するには、次の手順を実行します：
ユーザー > ユーザーグループに移動します。
各言語のユーザーグループを作成し、対応するグループにユーザーを追加します。
プラグインを複数回作成し、各言語ごとに1つずつ作成します。
必要な言語でコンテンツを提供し、ラベルを付けます。たとえば、スペイン語やポルトガル語で祝日や会社の休暇を追加できます。
プラグインの設定のためにギアホイールをクリックします。
上記のように、必要な言語でメニューの名前を提供します。
アプリ内の表示で、選択したユーザー、グループ、またはAPIトークンのためのオプションを選択します。
言語に応じてグループを選択します。
保存をクリックします。
プラグインのプレビューを使用して言語を確認します。
メニューに名前を複数の言語で提供する
複数の言語でコンテンツを作成するには、アプリがそのように構成されている必要があります。このオプションが欠けている場合は、アプリ管理者にお問い合わせください。
別の言語でコンテンツを追加したいプラグインに移動します。
設定のためにギアホイールをクリックします。
フィールドタイトルのためのドロップダウンメニューを開きます。
追加をクリックします。
コンテンツ言語を選択します。
ドロップダウンメニューには、アプリで有効になっているすべてのコンテンツ言語が含まれています。追加の言語が必要な場合は、管理者に有効にするよう依頼してください。
OKをクリックします。
新しい言語でタイトルを入力します。
タイトルはメニューの名前として使用されます。
保存をクリックします。
