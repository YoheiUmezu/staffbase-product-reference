# ユーザー名とパスワードによるオンボーディング

**URL**: https://support.staffbase.com/hc/ja/articles/360002484571
**取得日**: 2026-05-02 18:50:34 JST

## 内容

メールアドレスを使用できない場合、ユーザー名とパスワードを介してユーザーをオンボーディングすることは、アプリに全員を招待する素晴らしい方法です。
オンボーディングフェーズでは、ユーザーに連絡を取り、アクセスデータを配布します。このデータを直接提供することで、適切な手順を踏んで安全に保つことが重要です。オンボーディングフェーズ中にユーザーと個人的に連絡を取ることは、アプリの受け入れを大幅に高めるという利点があります。ユーザーへのアクセスデータの伝達の手間を減らすために、ユーザー名に姓を、ワンタイムパスワードに生年月日を使用するなど、よく知られた情報を利用することができます。
ユーザー名によるオンボーディングは、他のすべてのオンボーディング方法と並行して使用することもできます。
ユーザーデータの要件
ユーザーに次の情報を提供してください：
ユーザー名
ワンタイムパスワード — 特定の要件がある場合はワンタイムパスワードを作成するか、システムがワンタイムパスワードを生成します
以下の情報を提供することも強く推奨されます：
識別子 — CSVインポートなどのユーザー同期に必要
リカバリーコード — 設定するか、システムによって生成されるか、要求に応じて生成されるように空白のままにすることができます
名
姓
オプションで、次のような追加情報を提供することもできます：
所在地
部門
役職
カスタムプロファイルフィールドで使用したい追加情報
システムにユーザーデータを入力する
ユーザーがユーザー名でサインインできるようにするには、まずシステムにユーザーデータを作成する必要があります。ユーザーデータをインポートするには、いくつかのオプションから選択できます：
CSVインポート
ユーザーAPI
スタジオで手動でユーザーを作成
アプリへのユーザー招待
ユーザーデータをシステムにインポートします。たとえば、CSVインポートを介して。
システムにワンタイムパスワードとリカバリーコードを生成させる場合、ユーザーデータをエクスポートし、サインインデータを印刷してユーザーに渡します。
次の情報をユーザーに伝えます：
アプリの名前
アプリのダウンロード方法とウェブアプリへのリンクに関する情報
ユーザー名
ワンタイムパスワード
リカバリーコード
すべてのユーザーが安全な方法で個別の情報を受け取ることを確認してください！Staffbaseは、ユーザーの給与明細と一緒に配布することを推奨しています。
例：
親愛なるユーザー様、
アプリへようこそ！
Apple App StoreまたはGoogle Play Storeからアプリをダウンロードしてください。アプリはウェブアプリとしても利用可能です：app.yourcompany.de
以下はあなたの個人のサインインデータです：
ユーザー名：johnsmith
パスワード：XYZ1
リカバリーコード：456ABC
質問がある場合やサポートが必要な場合は、次のアドレスにご連絡ください：app-support@yourcompany.com。
よろしく、
アプリチーム！
ユーザーがアプリに登録する方法
ユーザーがアプリまたはウェブアプリを開きます。
ユーザーがユーザー名とワンタイムパスワードを入力します。
オプションで、ユーザーが利用規約またはプライバシーポリシーを確認します。
ユーザーが新しいパスワードを作成するように促されます。
オプションで、ユーザーがプロファイルを完成させ、メールアドレスなどの追加情報を入力するように求められます。
アカウント登録時にユーザーが提供したメールアドレスは、ユーザーの個人メールアドレスとして保存されます。 詳細はこちら。
この時点で、ユーザーはライブでアクティブなアカウントを持ち、ユーザー名と設定したばかりのパスワードを使用してサインインできます。ワンタイムパスワードは、最初のサインイン後は無効になります。ユーザーが自己定義したパスワードを忘れた場合、パスワードをリセットするためのリカバリーフローがあります。
ユーザーがパスワードをリセットする方法
忘れた場合、ユーザーはパスワードをリセットできます。設定できるオプションはいくつかあります。以下はユーザー名によるオンボーディングの標準オプションです：
ユーザーがアプリまたはウェブアプリを開きます。
ユーザーがリンクパスワードを忘れましたか？をクリックします。
ユーザーがリカバリーコードを入力します。
ユーザーが新しいパスワードを設定します。
