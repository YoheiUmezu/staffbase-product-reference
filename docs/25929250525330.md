# ブロッキング承認の使用

**URL**: https://support.staffbase.com/hc/ja/articles/25929250525330
**取得日**: 2026-05-02 19:02:54 JST

## 内容

アクティベーション
承認のブロックは、アサインされたレビュアーからの明示的な承認後にのみ投稿が公開されることを保証することで、組織が管理と一貫性を維持するのに役立ちます。
チャンネル内の承認者は、承認が要求されない限り、承認プロセスを経る必要はありません。彼らはすぐに投稿をスケジュールまたは公開することができます。
投稿が承認待ちのステータスを示している場合、まだ編集することができます。このステータスが下書きの投稿に表示されている場合、変更を加える前に承認リクエスターに相談してください。
承認のリクエスト
編集長
コンテンツの承認は、組織の基準に一貫性を持たせ、公開された投稿の品質を向上させます。公開タブ内で、責任者のユーザーにワンクリックで承認をリクエストできます。
Studioで、承認をリクエストしたいニュース投稿に移動します。
公開タブに移動します。
公開の下で、承認をリクエストをクリックします。
承認リクエストダイアログが開きます。
承認者ドロップダウンで、承認者を選択します。
リクエストごとに1人の承認者しか選択できません。
承認の締め切りを追加します。
オプションで、優先度を選択し、承認者にメッセージを追加します。
承認を送信をクリックします。
ユーザーに承認をリクエストしました。公開タブには、承認が保留中であることが表示されます。さまざまな承認ステータスについて詳しく学ぶ。
アサインされたユーザーは、承認リクエストについてのメールを受け取ります。メールにはニュース投稿へのリンクが含まれています。
アサインされたユーザーがリクエストを承認または拒否すると、メールで通知されます。
リクエストした承認の取り消し
編集長
承認者がまだ反応していない限り、承認リクエストを取り消すことができます。
自分が送信したリクエストのみを取り消すことができます。
Studioで、リクエストした承認を取り消したいニュース投稿に移動します。
公開タブに移動します。
リクエスト保留中の下で、リクエストをキャンセルをクリックします。
リクエストをキャンセルしますか？ダイアログが開きます。
私のリクエストをキャンセルをクリックします。
リクエストを取り消しました。承認者は、承認リクエストがキャンセルされたことを知らせるメールを受け取ります。いつでも新しい承認をリクエストできます。
リクエストが取り消された後に承認者が受け取ったメールのリンクをクリックすると、下書き投稿に移動しますが、リクエストは表示されません。
承認リクエストの承認または拒否
承認者としてアサインされた場合、承認リクエストに関するメールを受け取ります。メールにはニュース投稿へのリンクや、締切、優先度、コメントなどの情報が含まれています。

承認された投稿のスケジュールまたは公開
投稿が承認されると、スケジュールするか、すぐに公開することができます。
公開された投稿の編集
公開された投稿で、後から調整が必要な場合は、更新することができます。そのためには、まず投稿を非公開にする必要があります。
投稿を非公開にした後、再度公開するには承認が必要です。
Studioで、調整したい公開済みのニュース投稿に移動します。
公開タブに移動します。
公開の下で、非公開にするをクリックします。
投稿を非公開にしますか？ダイアログが開きます。
非公開にするをクリックします。
投稿を非公開にしました。これで、編集して再度公開するための承認をリクエストできます。
