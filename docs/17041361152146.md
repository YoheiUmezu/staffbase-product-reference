# Staffbaseプラットフォームにおけるユーザーのメールフィールドの種類

**URL**: https://support.staffbase.com/hc/ja/articles/17041361152146
**取得日**: 2026-05-02 18:53:37 JST

## 内容

この記事では、Staffbaseプラットフォームでユーザーが利用できるさまざまな種類のメールアドレスフィールドの概要を説明します。これらのメールフィールドは、サインイン、統合など、さまざまな目的で使用されます。プラットフォームで使用されるメールフィールドの種類は、組織の特定のニーズやプラットフォームの設定によって異なる場合があることに注意することが重要です。
ユーザーが持つことができるメールフィールドの種類は、以下の3つです：
プライマリメール
プライマリメールフィールドは、プラットフォームへのサインインプロセスに使用されるメールアドレスを保存します。これらは最も重要なメールアドレスの一部です。ユーザーは、Staffbaseプラットフォームでメール通知を受け取ったり、メールを通じてパスワードをリセットしたりするために、プライマリメールが必要です。このメールアドレスは、アカウント作成または登録時に定義されます。これはメタプロファイルフィールドであり、スタジオのプロファイルフィールドには表示されません。
プライマリメールアドレスは、以下のいずれかのオンボーディング方法を使用して定義できます：
アカウント作成
このメールアドレスは、ユーザーが以下のいずれかの方法でオンボードされるときに定義されます：
メールとパスワード：管理者がスタジオからメールアドレスを提供してユーザーを招待します。 詳細を学ぶ。
その他の方法：ITチームが以下のいずれかの方法でユーザーをオンボードする際に、プライマリメールアドレスをマッピングします：
SCIM
CSVインポート
ユーザーAPI
SSO
この場合、管理者はプライマリメールアドレスを確認および編集できます。 詳細を学ぶ。
アカウント登録
ユーザーは、アカウント作成時にメールアドレスが追加されていない場合、プラットフォームに初めて登録する際にメールアドレスの提供を求められます。
デフォルトでは、ユーザーはアカウント登録中に以下のいずれかの方法を使用して、オプションでメールアドレスの提供を求められます：
アクセスコードによるオンボーディング
ユーザー名とパスワードによるオンボーディング
このメールアドレスは、プライマリメールおよび個人メールの両方として機能します。ただし、このメールアドレスはスタジオに表示されず、ユーザーのアカウント詳細を編集する際に管理者には見えません。代わりに、「個人メールアドレス」というテキストが表示され、スタジオではフィールドが編集不可となります。
これは、GDPRポリシーに従ってユーザーのプライバシーを保護します。詳細については、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
個人メールは、ユーザーのアプリまたはイントラネットのアカウント設定の下に表示されます。 詳細を学ぶ。
特定のケースでは、登録プロセス中にユーザーは：
メールアドレスの提供を求められない
メールアドレスの提供を必須とされる
これは、プラットフォームの設定プロセス中に、Staffbaseテクニカルオンボーディングチームと共にこれらのケースを決定したためです。詳細については、Staffbaseサポートまたはカスタマーサクセスマネージャーにお問い合わせください。
プライマリメールの有無による異なるユースケースシナリオ
Staffbaseプラットフォームは非常に柔軟で、フロントラインおよびデスクトップの従業員に最適な方法で連絡を取るためのさまざまな方法を提供します。このため、プラットフォームは、プライマリメールアドレスの有無にかかわらず、ユーザーをサポートするオンボーディング方法の組み合わせになる可能性があります。
プライマリメールなし
その場合、登録プロセス中にメールアドレスの提供を求めないようにプラットフォームを設定できます。デスクトップ作業者には、SSO設定を提供できます。
プライマリメールの有無
プライマリメールアドレスを持つユーザーのサブセットと、プライマリメールアドレスを持たないユーザーのサブセットを持つこともできます。フロントライン作業者を1つの方法でオンボードし、デスクトップ作業者を別の方法でオンボードできます。
例えば、
デスクトップ作業者 - メールとパスワードによるオンボーディング
フロントライン作業者 - 登録中にメールのプロンプトなしでアクセスコードによるオンボーディング
パブリックメール
パブリックメールアドレスは、管理者が編集できるシステムプロファイルフィールドです。ユーザーは、システムプロファイルフィールドがユーザー編集可能に設定されている場合、自分のパブリックメールアドレスを編集することもできます。また、ユーザープロファイルに表示することもできます。
カスタムメール
カスタムメールアドレスは、入力タイプとしてメールを持つカスタムプロファイルフィールドに保存されます。ビジネスのニーズに応じて、必要なだけ多くのカスタムメールフィールドを作成できます。また、チームや部門のグループメールアドレスをユーザープロファイルに表示することもできます。例えば、sales@yourcompany.com
