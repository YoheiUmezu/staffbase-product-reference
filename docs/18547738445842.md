# ユーザー管理の向上のための新しいCSVインポート方法への移行

**URL**: https://support.staffbase.com/hc/ja/articles/18547738445842
**取得日**: 2026-05-02 18:58:03 JST

## 内容

Staffbaseは、ユーザーデータのインポートプロセスを簡素化し、効率化する新しいCSVインポート機能を提供します。この新機能は、従来のレガシー方式と比較して大幅な改善をもたらします。
この記事には以下が含まれます：
新機能の主な利点
新機能とレガシー機能の違い
新しいCSVインポートへの円滑な移行のためのベストプラクティス
Staffbaseは、この改善された方法をできるだけ早く採用することを推奨します。CSVインポートの利点を最大限に活用するためです。レガシー方式は将来的に廃止される予定であることに注意することが重要です。
改善された概要ページ
新しいインポート
新しい方法では、GDPRガイドラインに準拠した過去30日間に行われたインポートを表示する概要ページが提供されます。
概要ページには以下の詳細が含まれます：
インポートステータス
インポートしたユーザー
インポート日
データが更新されたユーザーの数
さらに、以下の機能を実行できます：
新しいインポート
インポートの開始
プレビューの表示
設定の編集
マッピングの編集
ファイルの削除
レガシーインポート
レガシー方式には概要ページがありますが、新しいインポート方式と比較して機能は制限されています。
レガシー方式では以下のことしかできません：
新しいインポートの作成
最後のインポートの詳細の表示
改善されたインポートワークフロー
新しいインポート
新しいインポートの改善点は以下の通りです：
新しいインポートワークフローは、異なるステップに分かれており、後で一時停止して再開することができます。これは、大規模なインポートを準備する際に特に便利です。また、インポートプロセス中にページから離れてもプロセスは停止しません。これにより、管理者はインポートがバックグラウンドで動作している間にスタジオで他のタスクに取り組むことができます。
複数の管理者が同時にページで作業できます。
CSVファイルの内容をプロファイルフィールドにマッピングできる改善されたユーザーインターフェース。
インポートを許可します：
選択したユーザー
このオプションは、CSVファイル内のユーザーのデータのみを更新します。プラットフォーム内の残りのユーザーは影響を受けません。このオプションは、レガシーインポート方式の部分的CSVインポートを置き換えます。
選択したユーザーをオフボードする場合は、CSVファイル内の各ユーザーのユーザーアカウントステータスを追加する列を追加する必要があります。 詳細を学ぶ。
すべてのユーザー
設定されたインポートの期待される変更をプレビューできます。
詳細を学ぶ。
レガシーインポート
レガシーCSVインポートの制限は以下の通りです：
複数の管理者が同時にページで作業できません。これは、新しいインポートを行う場合、現在のインポートプロセスが完了するかキャンセルされるまで進行できないことを意味します。
16MBまでのファイルのみをインポートします。
インポートを許可します：
すべてのユーザー
部分的CSVインポートタグを作成して選択したユーザー
この方法はスケーラブルではなく、部分的CSVインポートタグの管理と作業は難しいと見なされることがあります。
非同期インポート
新しいCSVインポート方法は非同期インポートをサポートしており、複数の管理者が同時にドラフトモードでファイルをアップロードしてインポートの準備を行うことができます。インポートプロセス中に、管理者はページから離れてもインポートに影響を与えません。これにより、管理者はインポートがバックグラウンドで動作している間にスタジオで他のタスクに取り組むことができます。さらに、インポートのステータスはいつでも確認できます。これにより、管理者はインポートがまだ実行中か完了しているかを理解するのに役立ちます。これは（レガシー）CSVインポート方法では不可能です。管理者がCSVインポートを即座に完了しない場合、他のすべての管理者は、インポートが完了するか、開始した管理者によって削除されるまで新しいインポートを開始することができません。
部分インポートと必須オープングループの代替オプション
新しいCSVインポート方法は、部分インポートと必須オープングループの同期をサポートしていません。これらの機能の代替手段については、引き続きお読みください。
部分インポート
新しいCSVインポート方法は、その高度な機能により、レガシー機能からの部分CSVインポートを無効にします。選択したユーザーを更新するオプションを部分インポートの代わりに使用できます。過去にグループ管理のために部分CSVインポートタグを使用していた場合、Staffbaseはタグを埋めるためのカスタムフィールドを作成することを提案します。次に、インポート中にカスタムフィールドに更新する必要があるユーザーの対応するタグ値を持つCSVファイルを作成できます。この方法で、カスタムフィールドをタグ付け可能に設定し、条件付きグループを構成するために使用できます。
必須オープングループ
必須グループを維持するためのCSVインポート方法の使用が少ないため、新しいCSVインポート方法は以下のサポートを提供していません：
必須グループにユーザーを追加する
必須グループからユーザーを削除する
過去に必須グループを管理するためにCSVインポートを使用していた場合、Staffbaseはオープングループと条件付きグループの組み合わせを使用して同じ結果を達成することを推奨します。
