# CSVインポートの設定

**URL**: https://support.staffbase.com/hc/ja/articles/19811653334930
**取得日**: 2026-05-02 18:59:35 JST

## 内容

有効化
管理者
この記事の情報は、従業員アプリまたはStaffbaseイントラネットを利用している場合、またはStaffbaseメールをアプリやイントラネットと一緒に利用している場合に適用されます。Staffbaseメールを単体製品として利用している場合は、StaffbaseメールでのCSVインポートの設定を参照してください。
CSVファイルをアップロードした後、インポートの準備を行うことができます。
設定を構成する
これらの手順は、すべてのユーザーまたは一部のユーザーをデルタインポート方式でインポートする場合のみ適用されます。一部のユーザーを部分インポートオプションでインポートする場合は、部分CSVインポートの概要を参照してください。
Studioでユーザー > CSVインポート > インポートしたいファイルに移動します。
3つのドットをクリックして、その他のオプションメニューを開きます。
インポート開始をクリックします。
下書きステータスのファイルのみ、インポート開始オプションが利用できます。
設定ページが開きます。
インポートタイプで、次のいずれかを選択します。
更新する
このオプションは、CSVファイルに含まれるユーザーのデータのみを更新します。プラットフォーム上の他のユーザーデータはそのまま残ります。
このオプションはデルタインポートとも呼ばれます。デルタインポートでユーザーをオフボードしたい場合は、各ユーザーのアカウントステータスを含む列をCSVファイルに追加する必要があります。詳細はこちら。
すべて上書き
このオプションは、CSVファイルおよびプラットフォーム上のすべてのユーザーを更新します。また、CSVファイルに含まれていないアクティブユーザーを無効にし、保留中のユーザーを削除します。
テキストエンコーディングドロップダウンで、CSVファイルで使用されているテキストエンコーディングの種類を選択します。
セル区切り文字フィールドに、次のいずれかを入力します。
セミコロン（;）
カンマ（,）
CSVファイルのデフォルトの区切り文字はセミコロン（;）です。アメリカなど一部の地域では、CSVファイルでカンマ（,）が使われる場合があります。CSVファイルの区切り文字が、インポート時に選択したものと一致していることを確認してください。
次をクリックします。
マッピングページが開きます。
マッピングを構成する
マッピングのステップでは、CSVの列名を、ユーザー管理構成に応じてプラットフォーム上の対応するプロフィール フィールドにマッピングできます。たとえば、アカウントステータス値を含むCSV列を「ステータス」にマッピングしてデルタインポートでユーザーをオフボードしたり、言語ロケールを含むCSV列を「言語」にマッピングして、ユーザーの希望言語でメール招待を送信したりできます。
マッピングページで、識別子をマッピングの下にある識別子のチェックボックスを選択します。
残りのCSV列名を、プラットフォーム上の対応するプロフィール フィールドにマッピングします。
必要に応じて、次のオプションを選択します。
リカバリーコードを自動生成しますか？：ユーザーのリカバリーコードを生成する場合
新規ユーザーにメールで招待または保留中のユーザーにメールで招待：メール招待を送信する場合
メールでユーザーを招待するオプションは、CSV列をメールアドレスにマッピングした場合のみ表示されます。
プレビューを作成をクリックします。
ファイルをインポートを直接クリックすることもできますが、Staffbaseでは、プレビューでユーザーベースへの更新内容を確認することを推奨しています。
ファイルのマッピングとプレビューの作成が完了しました。次にインポート内容を確認します。
