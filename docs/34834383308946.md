# クラシック編集者で作成されたページの移行

**URL**: https://support.staffbase.com/hc/ja/articles/34834383308946
**取得日**: 2026-05-02 19:01:47 JST

## 内容

ベータ
ページ移行を使用すると、クラシックエディターで作成されたページを1つずつ移動できます。この方法により、プロセスを完全に管理でき、新しいContent Designerへのスムーズな移行と運用の安定性を維持できます。
どのページが移行の準備ができているかを特定するために、2つの指標があります。
ページ概要の「Content Designer対応」タブには、移行可能なすべてのページがリストされています。
クラシックエディターで既存のページを開くと、そのページが移行可能かどうかを示すバナーが表示されます。これは、ページ内のすべての要素がサポートされており、Content Designerに正常に移行できることを意味します。
大規模な組織の場合、Staffbaseは段階的な移行を推奨します。
ページを移行する
ページの移行は、管理された2段階のプロセスで行います。まず、新しいContent Designerでページがどのように表示されるかをプレビューします。その後、移行を完了します。
移行が完了すると：
公開済みページは、今すぐLIVEバージョンに置き換えられます。
下書きはContent Designer内でも下書きのまま残ります。
オリジナルのクラシックページのバージョンの履歴には引き続きアクセスできます。ただし、クラシックバージョンは履歴ビューで完全には表示されません。
Studioでコンテンツ > ページ > 移行したいページに移動します。
ページエディターの上部でContent Designerに移動をクリックします。
ページがプレビューモードで開き、すべての要素がContent Designerでどのように表示されるかを確認できます。
Content Designerで下書きをレビューします。
移行をキャンセルしてクラシックエディターに戻ることもできます。
移行を完了をクリックします。
確定するダイアログが開きます。
移行を完了をクリックします。
ページがContent Designerで開きます。
クラシックエディターで作成されたページを移行しました。これで、Content Designerでページを編集し、保存、更新、または公開することができます。
