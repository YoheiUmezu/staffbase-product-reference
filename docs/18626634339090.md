# コレクションを使用するための役割

**URL**: https://support.staffbase.com/hc/ja/articles/18626634339090
**取得日**: 2026-05-02 18:56:48 JST

## 内容

マイファイルとファイルマネージャーの役割の違いを知っておくことは重要です。コレクションの追加と管理には、コンテンツ特有の役割で十分です。しかし、スタジオ全体でファイルやコレクションを削除する必要がある場合は、より強力な役割が必要です。

役割の詳細
ユーザー
ユーザーはコレクションにアクセスできません。
ローカル権限（例：ページ、チャンネル、またはスペースへの権限）を付与されたユーザーも、Studio にアクセスできます。その場合、これらのユーザーは Studio ユーザーと見なされ、Collections にアクセスできます。
チャンネル作成者 = 貢献者
チャンネル作成者 = 貢献者はコレクションにアクセスできません。
チャンネル編集者
チャンネル編集者はユーザーが1つのチャンネルを管理することを可能にする役割です。チャンネル編集者は、自分が作成したファイルや共有されたコレクションにアクセスできます。共有されていないファイルにはアクセスできません。自分が作成していないファイルや共有されていないコレクションを削除することはできません。
グループ管理者
グループ管理者はグループメンバーを追加および削除し、グループ設定を調整する権限を持っています。グループ管理者はコンテンツではなくユーザーに焦点を当てており、コレクションにはアクセスできません。
スペース管理者
スペース管理者はコンテンツ管理に関するスペース全体の管理権限を持ち、自分が作成したコレクションや共有されたコレクションにアクセスできます。共有されていないファイルにはアクセスできません。自分が作成していないファイルや共有されていないコレクションを削除することはできません。
システム全体の編集者
システム全体の編集者は、自分が作成したファイルや共有されたコレクションにアクセスできます。共有されていないファイルにはアクセスできません。自分が作成していないファイルや共有されていないコレクションを削除することはできません。
管理編集者
管理編集者は、自分が作成したファイルや共有されたコレクションにアクセスできます。共有されていないファイルにはアクセスできません。自分が作成していないファイルや共有されていないコレクションを削除することはできません。
管理者
管理者はシステム全体の管理権限を持ち、コンテンツ管理を含みます。彼らは共有されていないファイルやコレクションも表示できます。中央の場所からファイルやコレクションを削除することができます。
