Staffbaseプラットフォームメニューは、ユーザーを主要な機能やコンテンツに接続する主なナビゲーションツールです。メニューについて詳しく学ぶ。 Staffbaseは、モバイルとデスクトップ環境で異なるメニュータイプを提供しています。イントラネットユーザー向けには、水平メニューがメニューフライアウトとより深いナビゲーションレベルでコンテンツへの簡単なアクセスを提供します。水平メニューを有効にすると、デスクトップユーザーにのみ影響します。モバイルデバイスでは、ナビゲーションは垂直に配置されたままです。
イントラネットメニュー構造
イントラネットは、水平メニュー、またはトップナビゲーションとしても知られるものに十分なスペースを提供します。これはイントラネットの上部に水平に表示されます。メニュー項目は、ページ、ニュースチャンネル、プラグイン、フォルダで構成され、最大3層のフライアウトメニューを許可します。
フライアウトは、ユーザーが多数のフォルダをクリックせずに長いメニューパスを見ることができるため、有利です。フライアウトメニュー内の第1レベルの項目をクリックすると、メニューパスの画像と説明が表示され、第2レベルの項目に分岐します。第2レベルの項目にカーソルを合わせると、次のレベルが表示されます。最大3レベルのフライアウトは、デスクトップユーザーにシームレスなナビゲーション体験を提供し、メニュー内の項目を明確にすることでナビゲーションをより直感的にします。
注: フォルダ画像を選択する際には、フォルダ画像の推奨サイズ1000px x 400pxとフライアウトメニューの230px x 115pxを考慮してください。
水平メニューは、Staffbaseウェブアプリをイントラネットソリューションとして使用したり、HRプロセスやサードパーティツールへのアクセスなどのサービスを統合したりすることを望む組織に最適です。デバイスタイプに基づいて各メニュー項目の表示を設定することで、イントラネットメニューをユーザーベースに合わせて調整できます。 メニュー項目の表示設定について詳しく学ぶ。 デバイスの表示に関係なく、ユーザーはデフォルトでアクセスできるメニュー内のコンテンツのみを表示します。プラグイン自体でユーザーとグループのコンテンツ表示を設定すると、例えばチャットで、その設定が自動的にメニュー項目に適用されます。
公開エリアの垂直メニュー
サインインしたユーザー向けのイントラネットメニューに加えて、デスクトップ用の公開エリアメニューがあります。このメニューはウェブアプリインターフェースの側面に垂直に表示され、最大5レベルの深さを持つように構成できます。公開エリアについて詳しく学ぶ。
イントラネットメニューのベストプラクティス
デスクトップデバイスでは、従業員アプリがイントラネットの玄関口として機能し、より深いレベルのコンテンツにアクセスできます。デバイス画面が大きい場合、より深いレベルのフライアウトを備えた完全なコンテンツメニューにより、ユーザーはコンテンツをより早く見つけることができます。
トップレベルのナビゲーションには8つ以下のトピックを選択してください。
仕事関連のコンテンツとソーシャルコンテンツを分けてください。
コンテンツの重複を減らし、ユーザーがどのメニュー項目の下に情報があるかを推測しなくても済むようにしてください。
実装前にワークショップやユーザーテストでメニュー構造をテストしてください。
従業員アプリが組織と共に進化するように、メニュー構造を継続的にテストしてください。
